## 摘要
AI 记忆，特别是模型如何组织和检索历史消息，对大语言模型（LLMs）变得越来越重要。然而，现有方法（如 RAG 和 Graph-RAG）主要通过基于相似性的机制来检索记忆。虽然这种“系统 1”式的检索效率很高，但在需要全局推理或全面覆盖所有相关信息的场景中表现不佳。在这项工作中，我们提出了 Mnemis，这是一种新型记忆框架，它将“系统 1”相似性搜索与一种互补的“系统 2”机制（称为“全局选择”）相结合。Mnemis 将记忆组织成一个用于相似性检索的基础图和一个支持在语义层级上进行自顶向下、深思熟虑遍历的分层图。通过结合两种检索路线的互补优势，Mnemis 能够检索出在语义和结构上都相关的记忆项。Mnemis 在长期记忆基准测试中，在所有对比方法中均达到了最先进的性能，在使用 GPT-4.1-mini 时，在 LoCoMo 上得分为 93.9，在 LongMemEval-S 上得分为 91.6。

## Abstract
AI Memory, specifically how models organizes and retrieves historical messages, becomes increasingly valuable to Large Language Models (LLMs), yet existing methods (RAG and Graph-RAG) primarily retrieve memory through similarity-based mechanisms. While efficient, such System-1-style retrieval struggles with scenarios that require global reasoning or comprehensive coverage of all relevant information. In this work, We propose Mnemis, a novel memory framework that integrates System-1 similarity search with a complementary System-2 mechanism, termed Global Selection. Mnemis organizes memory into a base graph for similarity retrieval and a hierarchical graph that enables top-down, deliberate traversal over semantic hierarchies. By combining the complementary strength from both retrieval routes, Mnemis retrieves memory items that are both semantically and structurally relevant. Mnemis achieves state-of-the-art performance across all compared methods on long-term memory benchmarks, scoring 93.9 on LoCoMo and 91.6 on LongMemEval-S using GPT-4.1-mini.

*   **核心问题**：现有的长期记忆检索方法（如 RAG 和 Graph-RAG）主要依赖基于相似度的检索机制（类似于人类思维的“系统 1”/System-1）。这种方式在处理需要**全局推理**或**全面覆盖相关信息**的复杂查询时（例如“Dave 在 2023 年去过哪些城市？”）表现不佳，因为关键信息可能在语义上与查询词联系较弱，或者分散在海量历史记录中。
*   **研究背景**：随着 LLM 从文本生成器转向持久交互代理，如何高效组织和检索历史交互（情节记忆与语义记忆）成为关键。论文旨在引入一种类似于人类“系统 2”（System-2）的深思熟虑、自顶向下的检索机制，以弥补相似度检索的不足。

#### 2. 方法论：核心思想与关键技术
Mnemis 框架集成了两种存储组件和两条检索路线：
*   **基础图（Base Graph）**：存储原始情节（Episodes）、实体（Entities）、关系边（Edges）和情节边（连接实体与原始文本）。用于支持基于嵌入（Embedding）和关键词（BM25）的**系统 1 相似度搜索**。
*   **分层图（Hierarchical Graph）**：通过对实体进行自底向上的抽象，构建多层类别（Categories）。其构建遵循三个原则：
    1.  **最小概念抽象**：类别应保持足够的具体性以保留信息，同时具备概括性。
    2.  **多对多映射**：一个子节点可以属于多个父类别，反映实体的多维语义。
    3.  **压缩效率约束**：规定每个类别的最小子节点数及层级间的节点缩减比例，确保层级结构的平衡。
*   **双路线检索流程**：
    *   **路线 1 (System-1)**：通过向量相似度和 BM25 检索 Top-k 项，并使用 RRF（倒数排名融合）进行初步排序。
    *   **路线 2 (System-2 - 全局选择)**：LLM 从分层图的顶层开始，根据查询意图逐层选择相关的类别，最终定位到具体的实体及其关联的情节和边。
*   **最终整合**：将两条路线检索到的内容合并，通过重排序模型（Re-ranker）处理后，作为上下文输入给 LLM 生成答案。

#### 7. 优点（亮点）
*   **双系统启发**：将认知心理学的双过程理论成功应用于图检索，解决了传统 RAG 容易“迷失”在海量相似片段中的问题。

*   **鲁棒性**：通过消融实验证明，即使在较小的 Top-k 设置下，Mnemis 依然能保持较高的检索召回率。

*   **模态限制**：目前主要针对文本信息，尚未扩展到多模态记忆（如图像、视频交互历史）。
