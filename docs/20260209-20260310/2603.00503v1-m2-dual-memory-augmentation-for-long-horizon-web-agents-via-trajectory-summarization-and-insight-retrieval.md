## 摘要
基于多模态大语言模型（MLLMs）的智能体在自主网络导航中展现出巨大潜力。然而，处理长程（long-horizon）任务仍然是一个关键瓶颈。现有的策略通常严重依赖大规模数据收集和模型训练，但在面对复杂、长程场景时，仍面临高昂的计算成本和推理能力不足的问题。为此，我们提出了 M$^2$，这是一个无需训练的记忆增强框架，旨在优化上下文效率和决策鲁棒性。我们的方法结合了双层记忆机制，协同利用动态轨迹摘要（内部记忆）将冗长的交互历史压缩为简洁的状态更新，以及见解检索增强（外部记忆）通过从离线见解库中检索可操作的指南来引导智能体。在 WebVoyager 和 OnlineMind2Web 上的广泛评估表明，M$^2$ 持续超越基准模型，使 Qwen3-VL-32B 的成功率提升高达 19.6%，Token 消耗减少 58.7%，而像 Claude 这样的闭源模型在显著降低计算开销的同时，准确率提升了高达 12.5%。

## Abstract
Multimodal Large Language Models (MLLMs) based agents have demonstrated remarkable potential in autonomous web navigation. However, handling long-horizon tasks remains a critical bottleneck. Prevailing strategies often rely heavily on extensive data collection and model training, yet still struggle with high computational costs and insufficient reasoning capabilities when facing complex, long-horizon scenarios. To address this, we propose M$^2$, a training-free, memory-augmented framework designed to optimize context efficiency and decision-making robustness. Our approach incorporates a dual-tier memory mechanism that synergizes Dynamic Trajectory Summarization (Internal Memory) to compress verbose interaction history into concise state updates, and Insight Retrieval Augmentation (External Memory) to guide the agent with actionable guidelines retrieved from an offline insight bank. Extensive evaluations across WebVoyager and OnlineMind2Web demonstrate that M$^2$ consistently surpasses baselines, yielding up to a 19.6% success rate increase and 58.7% token reduction for Qwen3-VL-32B, while proprietary models like Claude achieve accuracy gains up to 12.5% alongside significantly lower computational overhead.

#### 1. 核心问题与整体含义
论文关注的核心问题是**多模态大语言模型（MLLM）智能体在处理长程（Long-Horizon）网页导航任务时的低效与失效**。
*   **研究背景**：现实中的网页任务通常需要数十步交互。主流的“全上下文”策略将所有历史截图和文本堆叠在 Prompt 中，导致两个痛点：
    1.  **计算成本高昂**：上下文窗口随步骤呈线性甚至超线性增长，消耗大量 Token。
    2.  **性能退化**：冗余信息导致模型出现“迷失在中间”（Lost-in-the-middle）现象，难以捕捉关键任务线索。
*   **研究动机**：现有的记忆增强方法多依赖昂贵的监督微调（SFT）或强化学习（RL）。本文旨在提出一种**无需训练（Training-free）**、轻量化且高效的记忆管理框架。

#### 2. 方法论
M$^2$ 框架引入了**双层记忆机制**，将决策上下文从无限增长的原始历史转变为精炼的结构化信息：
*   **内部记忆（Internal Memory, In-Mem）：动态轨迹摘要**
    *   **核心思想**：模仿人类认知，只保留高层抽象而非视觉细节。
    *   **技术细节**：在每一步交互中，要求智能体在输出动作的同时，生成一个简洁的文本摘要（包含当前页面状态和已执行动作）。
    *   **流程**：新生成的摘要会被追加到摘要链中，而之前的原始截图和冗长历史会被丢弃。这使得上下文长度随任务深度仅呈亚线性增长。
*   **外部记忆（External Memory, Ex-Mem）：见解检索增强**
    *   **核心思想**：利用跨任务的“专家经验”指导当前决策。
    *   **技术细节**：构建一个离线“见解库”（Insight Bank），从 5.5 万条成功轨迹中提取通用的 UI 交互规则（如：搜索策略、导航效率、状态验证逻辑）。
    *   **流程**：推理时，根据用户查询（Query）的语义相似度，利用 Sentence Transformer 检索 Top-i 个相关见解，并作为“防御性提示”注入系统 Prompt。

#### 3. 实验设计
*   **数据集/场景**：
    *   **WebVoyager**：涵盖 12 个代表性网站（如 Amazon, GitHub, Apple 等）。
    *   **OnlineMind2Web**：包含 291 个真实世界的在线任务，分为简单、中等、困难三个等级。
*   **Benchmark 与对比方法**：
    *   对比了各模型的 **Normal（全上下文）** 模式。
    *   对比了 SOTA 模型如 **UI-TARS-1.5** 和 **OpenAI CUA**。
*   **评估模型**：闭源模型 Claude-3.7-Sonnet、Claude-Sonnet-4；开源模型 Qwen3-VL-32B。

#### 4. 资源与算力
*   **算力设施**：实验在基于 Kubernetes (K8s) 的高并发集群上运行，部署了 **64 个并发节点**，每个节点托管一个虚拟计算机环境。
*   **训练时长**：该框架本身是**无需训练**的。
*   **离线处理**：使用 Ovis-inhouse 80A3B 模型作为摘要器来构建离线见解库。
*   **推理开销**：单次语义检索的延迟仅约 **6 毫秒**，相对于平均 122 秒的任务时长几乎可以忽略。

#### 5. 实验数量与充分性
*   **实验规模**：在两个大型基准测试上进行了广泛测试，涵盖了从电商到学术、地图等多种网页领域。
*   **消融实验**：非常充分。作者针对历史窗口大小（k）、相似度函数（TF-IDF vs. ST）、检索见解数量（i）以及内部记忆的组成成分（视觉 vs. 动作）分别做了对比实验。
*   **客观性**：使用 GPT-4o 作为第三方公正评判者，结合最终响应和轨迹截图进行成功率判定，减少了主观偏差。

#### 6. 主要结论与发现
*   **性能大幅提升**：在 WebVoyager 上，Qwen3-VL-32B 的成功率提升了 **16.2%**；在 OnlineMind2Web 的中等难度任务中，提升幅度高达 **22.3%**。
*   **成本显著降低**：Qwen3-VL-32B 的平均 Token 消耗减少了 **58.7%**，Claude 系列模型也减少了 **30%~55%**。
*   **开源追平闭源**：通过 M$^2$ 增强的开源 Qwen3-VL-32B（74.0%）在性能上超越了未经增强的 Claude-3.7-Sonnet（72.0%）。
*   **长程鲁棒性**：内部记忆有效解决了上下文爆炸问题，使智能体在长路径任务中表现更稳定，减少了因信息过载导致的逻辑中断。

#### 7. 优点
*   **训练无关性**：无需昂贵的 GPU 训练资源即可部署，具有极强的通用性和即插即用特性。
*   **双重维度优化**：内部记忆解决“效率”问题，外部记忆解决“鲁棒性/经验”问题，两者协同效应明显。
*   **跨任务迁移**：证明了从成功轨迹中提取的 UI 交互规则具有普适性，能有效指导不同网站的类似操作。

#### 8. 不足与局限
*   **冷启动开销**：在任务的第一步，由于注入了外部见解，Token 消耗会高于基准模式（虽然随步骤增加后总成本会大幅下降）。
*   **见解库依赖**：外部记忆的质量高度依赖于预先构建的见解库。如果面对全新的、交互逻辑迥异的网页，检索到的见解可能产生误导。
*   **静态见解风险**：网页 UI 频繁更新，离线提取的见解可能随时间失效，需要定期维护见解库。
