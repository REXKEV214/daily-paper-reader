## 摘要
大多数外部存储模块（External Memory Module）的评估都假设一个静态设置：存储是离线构建的，并在固定状态下进行查询。在实践中，存储是流式的：新事实不断到达，插入与检索交替进行，且存储状态在模型提供查询服务时不断演变。在这种模式下，准确性和成本受整个存储生命周期的支配，该周期涵盖了信息的摄取、维护、检索以及将其整合到生成过程中的全过程。我们提出了 Neuromem，这是一个可扩展的测试平台，用于在交替插入与检索的协议下对外部存储模块进行基准测试，并将其生命周期分解为五个维度，包括存储数据结构、归一化策略、整合策略、查询制定策略和上下文集成机制。利用 LOCOMO、LONGMEMEVAL 和 MEMORYAGENTBENCH 三个具有代表性的数据集，Neuromem 在共享的服务栈中评估了可互换的变体，并报告了 token 级 F1 分数以及插入/检索延迟。总体而言，我们观察到性能通常随着存储在多轮中的增长而下降，且与时间相关的查询仍然是最具挑战性的类别。存储数据结构在很大程度上决定了可达到的质量边界，而激进的压缩和生成式集成机制主要是在插入和检索之间转移成本，准确率提升有限。

## Abstract
Most evaluations of External Memory Module assume a static setting: memory is built offline and queried at a fixed state. In practice, memory is streaming: new facts arrive continuously, insertions interleave with retrievals, and the memory state evolves while the model is serving queries. In this regime, accuracy and cost are governed by the full memory lifecycle, which encompasses the ingestion, maintenance, retrieval, and integration of information into generation. We present Neuromem, a scalable testbed that benchmarks External Memory Modules under an interleaved insertion-and-retrieval protocol and decomposes its lifecycle into five dimensions including memory data structure, normalization strategy, consolidation policy, query formulation strategy, and context integration mechanism. Using three representative datasets LOCOMO, LONGMEMEVAL, and MEMORYAGENTBENCH, Neuromem evaluates interchangeable variants within a shared serving stack, reporting token-level F1 and insertion/retrieval latency. Overall, we observe that performance typically degrades as memory grows across rounds, and time-related queries remain the most challenging category. The memory data structure largely determines the attainable quality frontier, while aggressive compression and generative integration mechanisms mostly shift cost between insertion and retrieval with limited accuracy gain.

## 1. 核心问题与整体含义（研究动机和背景）
随着长文本和交互式 LLM 应用（如医疗助手、具身智能）的发展，**外部存储模块（External Memory Modules）**已成为核心组件。然而，现有的评估方法存在三大缺陷：
*   **静态评估局限性**：大多评估假设存储是离线构建后进行固定查询，忽视了实际应用中“新事实持续到达、插入与检索交替进行”的**流式（Streaming）**特征。
*   **黑盒系统评估**：现有研究多将存储系统视为整体，难以区分是哪个设计环节（如索引结构或检索策略）影响了最终的准确率和延迟。
*   **效率指标缺失**：缺乏对存储增长过程中维护开销、模型推理与存储模块开销之间归因的深入分析。

**Neuromem** 的提出旨在通过一个可扩展的测试平台，模拟真实的流式交互协议，并对存储生命周期进行细粒度分解，以揭示性能与成本之间的深层权衡。

## 2. 方法论：核心思想与关键技术
Neuromem 的核心思想是将外部存储的生命周期分解为**五个正交的设计维度（D1–D5）**，并基于**交替插入与检索协议**进行评估。

### 核心维度分解：
*   **D1：存储数据结构 (Memory Data Structure)**：存储的物理组织方式，分为分区式（如向量库、FIFO 队列）和层次式（如知识图谱）。
*   **D2：归一化策略 (Normalization)**：将原始上下文转化为可存储单元的过程（如保留原文、摘要提取、三元组重写）。
*   **D3：整合策略 (Consolidation)**：存储状态的动态维护，包括冲突解决（CRUD）、衰减淘汰（根据遗忘曲线删除）和结构精炼。
*   **D4：查询构建策略 (Query Formulation)**：将用户意图转化为检索信号（如关键词提取、查询分解）。
*   **D5：上下文集成机制 (Context Integration)**：将检索到的证据合成最终上下文（如重排序、多查询融合）。

### 算法流程：
1.  **流式协议**：按照时间戳对请求流进行排序。
2.  **插入流水线**：执行 `PRE_INS`（归一化）-> `INSERT`（写入 D1）-> `POST_INS`（整合维护）。
3.  **检索流水线**：执行 `PRE_RET`（查询构建）-> `SEARCH`（从 D1 检索）-> `POST_RET`（集成生成）。
4.  **背压机制**：在测试点阻塞数据流，确保检索是在特定历史状态下的准确评估。

## 6. 主要结论与发现
*   **[F1] 时间熵不可避免**：随着存储增长，所有系统的性能都会普遍下降（F1 下降约 13%-44%），且 LLM 驱动的维护策略难以完全抵消这种衰减。
*   **[F2] 数据结构决定上限**：混合索引（向量+倒排）通常构成性能的最优前沿，而下游优化无法弥补存储层的信息损失。
*   **[F3] 语义压缩是有损的**：将对话重写为结构化三元组（Rewrite）会导致性能崩溃（F1 下降 50% 以上），保留原始文本纹理效果更好。
*   **[F4] 生成式优化存在“延迟税”**：查询分解、多查询扩展等生成式步骤会增加数秒延迟，但准确率提升微乎其微，甚至为负。
*   **[F5] 启发式策略占优**：在流式场景下，基于热度或遗忘曲线的简单启发式维护策略在效率和效果上均优于昂贵的生成式维护。

## 7. 优点

*   **工程参考价值高**：揭示了许多“看似高级”的生成式优化在实际应用中性价比极低，具有很强的工业指导意义。

## 8. 不足与局限
*   **图存储扩展性挑战**：由于图维护（如 HippoRAG）的延迟随存储规模呈二次方增长，实验中不得不排除了一些复杂的迭代图精炼方法。

*   **短/长期存储界限模糊**：目前尚未能很好地定义和区分工作内存与长期存储之间的动态迁移界限。
*   **应用场景限制**：主要集中在文本对话，未涉及多模态或超大规模异构数据的流式存储。
