## 摘要
长程智能体通常将交互历史压缩为写入时摘要。这产生了一个根本性的“先写后查”障碍：压缩决策是在系统了解未来查询的关键点之前做出的。因此，摘要可能导致无法验证的遗漏——决定性的约束（例如过敏史）可能会被丢弃，导致智能体无法用可追溯的证据来证明其回答的合理性。保留原始日志可以恢复权威的事实来源，但默认基于原始日志进行检索成本高昂：许多查询可以通过摘要回答，而原始日志检索仍需处理长得多的上下文，从而增加了 Token 消耗和延迟。我们提出了 TierMem，这是一个溯源链接框架，将检索视为推理时的证据分配问题。TierMem 使用双层存储层级，以最廉价的充分证据进行回答：它默认查询快速摘要索引，并且仅在摘要证据不足时，由运行时充分性路由器升级到不可变的原始日志库。随后，TierMem 将验证后的结果作为新的摘要单元写回，并与其原始来源链接。在 LoCoMo 数据集上，TierMem 实现了 0.851 的准确率（对比仅使用原始日志的 0.873），同时减少了 54.1% 的输入 Token 和 60.7% 的延迟。

## Abstract
Long-horizon agents often compress interaction histories into write-time summaries. This creates a fundamental write-before-query barrier: compression decisions are made before the system knows what a future query will hinge on. As a result, summaries can cause unverifiable omissions -- decisive constraints (e.g., allergies) may be dropped, leaving the agent unable to justify an answer with traceable evidence. Retaining raw logs restores an authoritative source of truth, but grounding on raw logs by default is expensive: many queries are answerable from summaries, yet raw grounding still requires processing far longer contexts, inflating token consumption and latency.   We propose TierMem, a provenance-linked framework that casts retrieval as an inference-time evidence allocation problem. TierMem uses a two-tier memory hierarchy to answer with the cheapest sufficient evidence: it queries a fast summary index by default, and a runtime sufficiency router Escalates to an immutable raw-log store only when summary evidence is insufficient. TierMem then writes back verified findings as new summary units linked to their raw sources. On LoCoMo, TierMem achieves 0.851 accuracy (vs.0.873 raw-only) while reducing input tokens by 54.1\% and latency by 60.7%.

## 2. 方法论
TierMem 采用了一种受计算机缓存体系结构启发的**双层溯源感知存储架构**：

### 核心架构
1.  **Tier-2（原始日志层，$M_R$）**：存储不可变的、分页的原始对话记录（如每页 1000 Token），作为权威的事实来源。
2.  **Tier-1（溯源索引层，$M_S$）**：存储紧凑的摘要及其向量嵌入，每个摘要单元都包含指向 Tier-2 对应页面的**溯源指针（Provenance Links, $\rho$）**。

### 关键技术流程
*   **动态路由（Adaptive Routing）**：引入轻量级路由器 $\pi_\theta$。对于每个查询，首先检索 Tier-1。路由器判断现有摘要是否“证据充足”：
    *   **ANSWER（命中）**：若充足，直接基于摘要生成回答（低延迟、低成本）。
    *   **ESCALATE（未命中）**：若不足，触发“升级”机制，利用溯源指针精准定位原始日志页面，并进行有限步数的深度搜索。
*   **验证写回（Verified Write-Back）**：在升级路径中获取的原始证据，经验证后提炼为新的摘要单元写回 Tier-1，并保留溯源链接，以便未来类似查询能直接从 Tier-1 获益。
*   **路由器优化**：采用两阶段训练：
    1.  **SFT（监督微调）**：从 GPT-5 蒸馏思维链（CoT）推理过程。
    2.  **GRPO（群体相对策略优化）**：通过强化学习平衡准确率、Token 成本和“误报”惩罚。

## 7. 优点
*   **设计巧妙**：通过分层存储模拟了计算机系统的缓存机制，解决了 LLM 长上下文处理的经济性问题。
*   **可审计性强**：所有生成的回答都可以追溯到不可变的原始日志，极大地增强了智能体的可靠性。
*   **闭环进化**：验证写回机制允许系统在运行过程中不断优化其“快速路径”的覆盖范围。

## 8. 不足与局限
*   **路由器严谨性**：实验显示仍存在约 34.3% 的错误源于“虚假缓存命中”（即路由器错误地认为摘要足够），说明轻量级路由器的判断力仍有提升空间。
*   **下游依赖**：如果原始日志本身未记录关键信息，或者升级路径中的检索器失效，系统仍无法给出正确答案。
*   **工程复杂度**：相比于简单的向量数据库，维护溯源链接和分页日志增加了系统的实现和维护难度。
*   **实时性挑战**：虽然平均延迟降低，但对于需要“升级”的查询，其单次响应时间仍然较长。
