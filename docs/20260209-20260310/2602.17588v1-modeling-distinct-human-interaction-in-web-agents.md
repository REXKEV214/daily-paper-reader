## 摘要
尽管自主 Web 智能体取得了快速进展，但在任务展开过程中，人类的参与对于塑造偏好和纠正智能体行为仍然至关重要。然而，目前的智能体系统缺乏对人类何时以及为何干预的原则性理解，通常在经过关键决策点时仍保持自主运行，或者请求不必要的确认。在这项工作中，我们引入了人类干预建模任务，以支持协作式 Web 任务执行。我们收集了 CowCorpus，这是一个包含 400 条真实用户 Web 导航轨迹的数据集，其中包含超过 4,200 个交替的人类和智能体动作。我们识别出四种截然不同的用户与智能体交互模式：放手式监督 (hands-off supervision)、介入式监察 (hands-on oversight)、协作式任务解决 (collaborative task-solving) 以及完全用户接管 (full user takeover)。利用这些见解，我们训练语言模型 (LM) 根据用户的交互风格预测其可能干预的时机，与基础语言模型相比，干预预测准确率提高了 61.4-63.4%。最后，我们将这些具备干预感知能力的模型部署在实时的 Web 导航智能体中，并在用户研究中对其进行评估，发现用户评价的智能体有用性提升了 26.5%。总之，我们的结果表明，对人类干预进行结构化建模可以产生更具自适应性和协作性的智能体。

## Abstract
Despite rapid progress in autonomous web agents, human involvement remains essential for shaping preferences and correcting agent behavior as tasks unfold. However, current agentic systems lack a principled understanding of when and why humans intervene, often proceeding autonomously past critical decision points or requesting unnecessary confirmation. In this work, we introduce the task of modeling human intervention to support collaborative web task execution. We collect CowCorpus, a dataset of 400 real-user web navigation trajectories containing over 4,200 interleaved human and agent actions. We identify four distinct patterns of user interaction with agents -- hands-off supervision, hands-on oversight, collaborative task-solving, and full user takeover. Leveraging these insights, we train language models (LMs) to anticipate when users are likely to intervene based on their interaction styles, yielding a 61.4-63.4% improvement in intervention prediction accuracy over base LMs. Finally, we deploy these intervention-aware models in live web navigation agents and evaluate them in a user study, finding a 26.5% increase in user-rated agent usefulness. Together, our results show structured modeling of human intervention leads to more adaptive, collaborative agents.

### 论文总结：建模 Web 智能体中不同的人类交互模式

#### 1. 核心问题与整体含义（研究动机和背景）
尽管自主 Web 智能体（如基于 LLM 的导航助手）发展迅速，但在处理复杂任务时仍需人类干预以纠正错误或对齐偏好。当前系统的主要痛点在于：**缺乏对人类干预时机和动机的理解**。这导致智能体要么在错误路径上盲目自主，要么频繁发出不必要的确认请求，增加了用户的监察负担（Oversight Burden）。本研究旨在通过建模人类干预行为，使智能体能够预判用户何时会介入，从而实现更高效、更具自适应性的人机协作。

#### 2. 方法论
论文提出了一套从数据收集到模型训练的完整框架：
*   **任务形式化**：将人机协作建模为部分可观测马尔可夫决策过程（POMDP）。干预预测被定义为一个步进式二元分类任务：在给定历史轨迹 $\tau_{t-1}$、当前观测 $o_t$ 和智能体拟采取动作 $\hat{a}_t$ 的情况下，预测用户是否会干预 $y_t \in \{0, 1\}$。
*   **交互模式识别**：通过对用户行为特征（干预频率、强度、位置、交还率）进行聚类分析（K-means），识别出四种风格：
    1.  **Takeover（接管型）**：干预频率低，通常在任务末端介入且不再交还控制权。
    2.  **Hands-on（介入型）**：频繁且高强度干预，但在介入后会交还控制权。
    3.  **Hands-off（放手型）**：几乎不干预，完全依赖智能体。
    4.  **Collaborative（协作型）**：选择性干预，干预时间短且迅速交还控制权。
*   **模型训练**：使用多模态大模型（LMM），输入包括网页截图、可访问性树（AXTree）和动作历史。通过监督微调（SFT）训练模型生成 `<ask_user>` 或 `<agent_continue>` 标记。
*   **关键指标 PTS**：提出了“完美时机得分”（Perfect Timing Score, PTS），该指标不仅衡量准确率，还通过时间距离惩罚（$L_2$ 距离）来评估预测干预时机的精确度。

#### 3. 实验设计
*   **数据集 (CowCorpus)**：包含 400 条真实用户轨迹，涉及 4,200 多个动作步骤。任务分为两类：
    *   **标准任务**：从 Mind2Web 基准中筛选的 10 个跨领域任务。
    *   **自由任务**：用户根据个人需求自定义的 10 个任务（如购物、信息检索、安全测试等）。

*   **用户研究**：将干预感知模型集成到 Chrome 扩展程序 **PlowPilot** 中，邀请原始用户进行实测评估。

*   **客观性**：通过在标准基准（Mind2Web）和真实自由任务上同时进行测试，并引入人类主观评分（Likert 量表），确保了评估的全面性。

#### 6. 主要结论与发现
*   **微调胜过规模**：经过 CowCorpus 微调的较小模型（如 Llava 8B）在干预预测时机（PTS）上优于未微调的巨型闭源模型（如 GPT-4o）。
*   **推理的副作用**：强制模型进行显式推理反而会降低 PTS 得分，因为干预往往是直觉性的，推理会导致模型过于保守或迟疑。
*   **风格适配有效**：针对特定交互风格（如 Hands-on）训练的模型在对应用户群中表现更好。
*   **实用性提升**：在实时部署中，干预感知模型减少了不必要的打扰，使用户感知的智能体有用性提升了 **26.5%**。

#### 7. 优点
*   **视角独特**：从“建模人类行为”而非单纯“提升智能体自主性”的角度切入，更符合真实协作场景。
*   **指标创新**：PTS 指标解决了二元分类无法衡量时间偏移的问题，对实时交互系统具有参考价值。
*   **数据真实**：CowCorpus 捕捉了真实网页上的动态交互，而非静态模拟环境。

#### 8. 不足与局限
*   **样本量限制**：最后的实时部署用户研究仅有 4 名参与者，样本量较小，结论可能存在偏差。
*   **数据稀疏性**：对于“Takeover”等干预样本极少的类别，模型难以进行有效学习。
*   **计算延迟**：虽然模型能预测干预，但 LLM 本身的推理延迟（平均每步 70-90 秒）仍是制约协作体验的瓶颈。
*   **泛化风险**：数据集主要集中在 20-30 岁用户，可能无法完全代表所有年龄段或技术背景的人群。
