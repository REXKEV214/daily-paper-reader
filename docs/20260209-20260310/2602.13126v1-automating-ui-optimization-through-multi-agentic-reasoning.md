## 摘要
我们提出了 AutoOptimization，这是一种用于自适应用户界面的新型多目标优化框架。根据用户对修改 UI 的口头偏好，我们的框架在候选布局上引导基于优先级的帕累托前沿（Pareto frontier）搜索。它为 UI 布局选择合适的目标函数，同时根据用户的指令对其进行参数化，从而定义优化问题。随后，求解器生成一系列最优 UI 布局，我们的框架根据用户指令对这些布局进行验证，并使用最终方案对 UI 进行调整。因此，我们的方法克服了以往需要人工检查布局以及对目标参数使用群体平均值的局限性。我们在框架中按顺序集成了多个智能体，使系统能够利用它们的推理能力来解释用户偏好、配置优化问题并验证优化结果。

## Abstract
We present AutoOptimization, a novel multi-objective optimization framework for adapting user interfaces. From a user's verbal preferences for changing a UI, our framework guides a prioritization-based Pareto frontier search over candidate layouts. It selects suitable objective functions for UI placement while simultaneously parameterizing them according to the user's instructions to define the optimization problem. A solver then generates a series of optimal UI layouts, which our framework validates against the user's instructions to adapt the UI with the final solution. Our approach thus overcomes the previous need for manual inspection of layouts and the use of population averages for objective parameters. We integrate multiple agents sequentially within our framework, enabling the system to leverage their reasoning capabilities to interpret user preferences, configure the optimization problem, and validate optimization outcomes.

### 2. 方法论
AutoOptimization 框架由四个顺序协作的阶段组成：
*   **歧义检测 (Ambiguity Detection)**：第一个智能体分析用户指令、当前 UI 状态和环境截图。如果指令模糊（如未指定具体组件或位置），它会主动向用户提问，直到获得足够信息。
*   **配置 (Configuration)**：第二个智能体将细化后的指令翻译成数学优化问题。它动态选择目标函数（如视野、颈部疲劳、物理锚定、遮挡避让等）并估计相关参数（如交互频率、观察概率）。
*   **优化 (Optimization)**：计算求解器（使用 NSGA-III 算法）在定义的搜索空间内运行，生成一组代表不同权衡取舍的帕累托最优布局候选集。
*   **验证 (Validation)**：第三个智能体评估所有候选布局的截图，对比用户的原始意图，最终选出一个最符合要求的布局应用到界面上。

### 7. 优点
*   **全流程自动化**：首次实现了从自然语言输入到优化配置再到结果验证的闭环，无需人工干预优化参数。
*   **多智能体协作**：将复杂任务分解为检测、配置、验证三个环节，利用了 VLM 的推理长处，同时保留了数学优化求解器的严谨性。
*   **上下文感知**：能够结合物理环境（如避开显示器、利用桌面）进行语义化的 UI 布局。

### 8. 不足与局限
*   **指令类型限制**：目前主要支持“目标空间”指令（如“不要遮挡电视”），尚不支持精确的“设计空间”指令（如“向左移动 5 厘米”）。
*   **帕累托前沿假设**：框架假设用户的理想布局一定存在于数学定义的帕累托前沿上，如果目标函数定义不全，可能漏掉用户真正想要的解。
*   **延迟问题**：依赖云端 VLM 调用，总响应时间（约 20 秒）对于需要即时反馈的交互场景来说仍然偏长。
*   **样本偏差**：用户研究样本量（12人）相对较小，且主要为大学生，可能无法完全代表所有年龄段的用户群体。
