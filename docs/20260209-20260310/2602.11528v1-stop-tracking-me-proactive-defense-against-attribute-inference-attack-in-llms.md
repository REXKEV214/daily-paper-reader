## 摘要
最近的研究表明，大语言模型（LLM）可以从在线分享的用户生成文本中推断出用户的私密属性（如年龄、位置、性别），从而导致快速且大规模的隐私泄露。现有的基于匿名化的防御方法是粗粒度的，在匿名化泄露隐私的元素时缺乏词级精度。此外，这些方法具有固有的局限性，因为即使修改用户文本以隐藏敏感线索，模型的推理能力仍然可以实现属性推断。为了解决这些局限性，我们提出了一个统一的防御框架，将细粒度匿名化（TRACE）与防止推断的优化（RPS）相结合。TRACE 利用注意力机制和推理链生成来识别并匿名化泄露隐私的文本元素，而 RPS 采用轻量级的两阶段优化策略来诱导模型产生拒绝行为，从而防止属性推断。在多种 LLM 上的评估表明，TRACE-RPS 在开源模型上将属性推断的准确率从约 50% 降低到 5% 以下。此外，我们的方法具有强大的跨模型泛化能力、提示词变体鲁棒性以及效用与隐私之间的权衡。我们的代码可在 https://github.com/Jasper-Yan/TRACE-RPS 获取。

## Abstract
Recent studies have shown that large language models (LLMs) can infer private user attributes (e.g., age, location, gender) from user-generated text shared online, enabling rapid and large-scale privacy breaches. Existing anonymization-based defenses are coarse-grained, lacking word-level precision in anonymizing privacy-leaking elements. Moreover, they are inherently limited as altering user text to hide sensitive cues still allows attribute inference to occur through models' reasoning capabilities. To address these limitations, we propose a unified defense framework that combines fine-grained anonymization (TRACE) with inference-preventing optimization (RPS). TRACE leverages attention mechanisms and inference chain generation to identify and anonymize privacy-leaking textual elements, while RPS employs a lightweight two-stage optimization strategy to induce model rejection behaviors, thereby preventing attribute inference. Evaluations across diverse LLMs show that TRACE-RPS reduces attribute inference accuracy from around 50\% to below 5\% on open-source models. In addition, our approach offers strong cross-model generalization, prompt-variation robustness, and utility-privacy tradeoffs. Our code is available at https://github.com/Jasper-Yan/TRACE-RPS.

#### 1. 核心问题与整体含义
*   **研究背景**：大语言模型（LLM）具备强大的推理能力，能够从看似平常的用户生成文本（如社交媒体评论）中推断出用户的敏感隐私属性（如年龄、性别、地理位置、收入水平等）。
*   **核心问题**：这种“属性推断攻击”（Attribute Inference Attack, AIA）具有隐蔽性，不触发传统的安全过滤器。现有的防御手段（如基于规则的匿名化）往往过于粗粒度，且无法阻止模型通过逻辑推理从修改后的文本中挖掘残余信息。
*   **研究动机**：开发一种主动防御框架，在用户分享文本之前对其进行处理，既能精准抹除隐私线索，又能通过优化手段诱导模型拒绝执行推断任务。

#### 2. 方法论
论文提出了一个名为 **TRACE-RPS** 的统一防御框架，包含两个核心组件：
*   **TRACE（基于注意力和推理链的文本修订）**：
    *   **隐私词汇提取**：利用预训练模型的注意力机制，识别对属性推断贡献最大的特定词汇（Top-K 词汇）。
    *   **推理链引导匿名化**：模拟攻击者的推理过程，生成“推理链”（Inference Chain），识别文本中支持推断的逻辑步骤，并引导匿名化模型进行针对性修改。
*   **RPS（拒绝导向的扰动搜索）**：
    *   **核心思想**：在用户文本后添加一个轻量级的优化后缀（Suffix），改变模型的输出分布。
    *   **两阶段优化**：第一阶段（锚定阶段）优化后缀使模型首个 Token 输出为“I”；第二阶段（塑造阶段）优化后缀使后续 Token 输出为拒绝词（如“cannot”, “apologize”）。
*   **MPS（误导导向的扰动搜索）**：针对极强指令遵循能力、不愿拒绝的模型（如 Qwen），通过优化后缀将预测结果重定向至错误的属性值。

#### 6. 主要结论与发现
*   **防御效果显著**：在开源模型上，TRACE-RPS 将属性推断准确率从约 **50% 降低至 5% 以下**（部分接近 0%）。
*   **闭源模型适用性**：对于无法获取 Logits 的闭源模型（如 GPT-4o），仅使用 TRACE 匿名化也能显著优于现有基准方法。
*   **泛化能力强**：在一个模型上优化的 RPS 后缀可以有效地迁移到其他架构或规模的模型上。
*   **效用平衡**：在大幅提升隐私保护的同时，TRACE 能够保持较高的文本语义相似度和实用性（Utility）。

#### 7. 优点
*   **细粒度精准性**：相比传统的粗暴掩码，TRACE 利用注意力机制实现了词级的精准定位。
*   **主动性与独立性**：防御发生在用户侧，不依赖于模型服务商的安全对齐。
*   **两阶段优化创新**：RPS 的两阶段搜索策略（锚定+塑造）比单步优化更易收敛且效果更稳健。
*   **多场景覆盖**：同时考虑了拒绝推理（RPS）和误导推理（MPS）两种策略。

#### 8. 不足与局限
*   **白盒/灰盒依赖**：RPS 优化需要访问模型的 Logits，对于完全黑盒且无法获取概率分布的 API 服务，其优化效果受限（只能依靠 TRACE）。
*   **计算开销**：虽然比同类方法快，但对每个用户文本进行 10,000 次迭代优化在实时交互场景下仍有一定延迟。
*   **自适应攻击风险**：虽然论文测试了一些自适应攻击，但高级攻击者若已知晓 RPS 机制，可能会开发更复杂的预处理手段来剥离防御后缀。
