## 摘要
随着大语言模型参与扩展推理任务，它们在上下文窗口中积累了大量的状态——包括架构映射、权衡决策、代码库规范。当会话达到上下文限制并进行有损压缩时，这些理解就会丢失。我们提出了上下文内存虚拟化（CMV），这是一个将积累的 LLM 理解视为版本控制状态的系统。借鉴操作系统虚拟内存的概念，CMV 将会话历史建模为有向无环图（DAG），并具有正式定义的快照（snapshot）、分支（branch）和修剪（trim）原语，从而实现跨独立并行会话的上下文重用。我们引入了一种三阶段结构化无损修剪算法，该算法逐字保留每条用户消息和助手响应，同时通过剥离机械冗余（如原始工具输出、base64 图像和元数据），使 Token 数量平均减少 20%，对于具有显著开销的会话，减少量最高可达 86%。对 76 个真实世界编程会话进行的单用户案例研究评估表明，在提示词缓存（prompt caching）下，修剪在经济上仍然可行，其中在混合工具使用会话中收益最显著，平均减少 39%，并在 10 轮内达到盈亏平衡。参考实现在 https://github.com/CosmoNaught/claude-code-cmv 可用。

## Abstract
As large language models engage in extended reasoning tasks, they accumulate significant state -- architectural mappings, trade-off decisions, codebase conventions -- within the context window. This understanding is lost when sessions reach context limits and undergo lossy compaction. We propose Contextual Memory Virtualisation (CMV), a system that treats accumulated LLM understanding as version-controlled state. Borrowing from operating system virtual memory, CMV models session history as a Directed Acyclic Graph (DAG) with formally defined snapshot, branch, and trim primitives that enable context reuse across independent parallel sessions. We introduce a three-pass structurally lossless trimming algorithm that preserves every user message and assistant response verbatim while reducing token counts by a mean of 20% and up to 86% for sessions with significant overhead by stripping mechanical bloat such as raw tool outputs, base64 images, and metadata. A single-user case-study evaluation across 76 real-world coding sessions demonstrates that trimming remains economically viable under prompt caching, with the strongest gains in mixed tool-use sessions, which average 39% reduction and reach break-even within 10 turns. A reference implementation is available at https://github.com/CosmoNaught/claude-code-cmv.

### 论文总结：上下文内存虚拟化（CMV）

#### 1. 核心问题与整体含义（研究动机和背景）
在长期的 LLM 智能体（如编码助手）会话中，模型会在上下文窗口内积累大量的“状态信息”（如架构设计、决策权衡、代码规范）。然而，当上下文达到 Token 限制时，现有的系统（如 Claude Code）通常采用**有损压缩（Lossy Compaction）**，将数万 Token 的详细理解简化为几行摘要，导致模型丢失了辛苦建立的深度理解。
论文提出了**上下文内存虚拟化（CMV）**，旨在解决“上下文瞬时性”问题。它借鉴操作系统的虚拟内存概念，将对话历史视为可版本控制的状态，允许用户像使用 Git 一样对上下文进行快照、分支和重用，从而避免重复构建上下文的计算和经济成本。

#### 2. 方法论：核心思想与关键技术
CMV 系统由两个核心组件组成：

*   **基于 DAG 的状态模型**：
    *   将对话历史建模为**有向无环图（DAG）**。
    *   **节点（Node）**：不可变的会话快照（JSONL 格式）。
    *   **边（Edge）**：代表从某一快照派生出的独立工作分支。
    *   **核心原语**：包括 `Snapshot`（创建快照）、`Branch`（从快照启动新会话）、`Trim`（修剪并分支）和 `Tree`（可视化谱系）。
*   **三阶段结构化无损修剪算法（Three-Pass Trimming）**：
    *   **第一阶段（边界检测）**：快速扫描以识别原生压缩边界，跳过已被摘要化的历史内容。
    *   **第二阶段（工具 ID 收集）**：收集预边界阶段的工具调用 ID，用于处理“孤儿”结果。
    *   **第三阶段（流式过滤）**：应用修剪规则。**核心原则是“结构化无损”**，即逐字保留所有用户消息和助手响应，但剔除机械性冗余：
        *   截断超过阈值（默认 500 字符）的原始工具输出（如长文件读取内容）。
        *   移除 Base64 图像块和不可移植的“思考块”（Thinking blocks）。
        *   **孤儿处理**：自动移除那些因历史截断而失去对应调用指令的工具结果，确保符合 API 校验架构。

#### 3. 实验设计
*   **数据集/场景**：使用了作者本人在三个月内通过 Claude Code（使用 Claude Opus 4.6 模型）产生的 **76 个真实世界编码会话**。

    *   引入了**经济成本模型**，对比在“提示词缓存（Prompt Caching）”机制下，修剪带来的 Token 减少收益是否能抵消因缓存失效（Cache Miss）产生的冷启动惩罚。

#### 5. 实验数量与充分性
*   **实验规模**：共 76 组真实会话，涵盖了从 5,000 到 132,000 Token 不等的上下文长度。
*   **充分性评价**：
    *   **优点**：实验基于真实、长期的编码任务，具有很高的实用参考价值；详细分析了盈亏平衡点（Break-even），考虑了缓存机制的复杂影响。
    *   **局限**：属于**单用户案例研究**，样本量虽然在会话维度尚可，但缺乏多用户、多任务类型的多样性验证。

#### 6 主要结论与发现
*   **Token 减量显著**：平均减少 20% 的 Token，在工具密集型会话中平均减少 39%，最高减量达 86%。
*   **经济可行性**：尽管修剪会破坏提示词缓存，但在混合工具场景下，平均只需 10 轮对话即可通过节省的 Token 成本达到盈亏平衡。
*   **上下文重用价值**：CMV 的最大价值在于避免了“上下文重建”的成本（平均每会话 84k Token），通过分支功能，模型可以瞬间获得之前的架构理解。

#### 7. 优点
*   **Git 式工作流**：为 LLM 开发引入了版本控制范式，支持并行探索不同的技术路径。
*   **结构化无损**：相比语义压缩（如摘要），该方法不改变模型原始的推理记录，降低了幻觉风险。
*   **API 兼容性**：通过三阶段设计解决了工具调用 ID 匹配的硬性约束，具有很强的工程落地性。

#### 8. 不足与局限
*   **语义盲目性**：修剪是基于结构的，算法无法判断某个被截断的工具输出是否对后续推理至关重要（虽然模型可以通过重新读取文件来弥补）。
*   **评估偏差**：使用字节到 Token 的启发式估算（chars/4）可能在处理图像密集型会话时产生偏差。
*   **应用限制**：目前主要针对编码智能体优化，对于纯创意写作或非工具类任务的收益可能较小。
