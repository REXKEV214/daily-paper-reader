## 摘要
大语言模型（LLMs）及其基于此的对话代理在预训练和用户交互过程中会接触到个人数据（PD）。先前研究表明 PD 可能会重新出现，但用户对于模型将特定信息与其身份关联的强度缺乏了解。我们对八个 LLM（包括 3 个开源模型和 5 个基于 API 的模型，如 GPT-4o）进行了 PD 审计，并引入了 LMP2（语言模型隐私探测器），这是一种经过两项形成性研究（N=20）优化的、以人为中心且保护隐私的审计工具。我们针对欧盟居民开展了两项研究，旨在捕捉：(i) 用户对 LLM 生成 PD 的直觉（N1=155），以及 (ii) 用户对工具输出的反应（N2=303）。实证结果显示，模型能够为知名人士自信地生成多个类别的 PD。对于普通用户，GPT-4o 生成的 11 个特征（如性别、发色、语言）准确率达到 60% 或更高。最后，72% 的参与者希望控制模型生成的与其姓名相关的关联，这引发了关于 PD 的界定以及数据隐私权是否应延伸至 LLM 的思考。

## Abstract
Large language models (LLMs), and conversational agents based on them, are exposed to personal data (PD) during pre-training and during user interactions. Prior work shows that PD can resurface, yet users lack insight into how strongly models associate specific information to their identity. We audit PD across eight LLMs (3 open-source; 5 API-based, including GPT-4o), introduce LMP2 (Language Model Privacy Probe), a human-centered, privacy-preserving audit tool refined through two formative studies (N=20), and run two studies with EU residents to capture (i) intuitions about LLM-generated PD (N1=155) and (ii) reactions to tool output (N2=303). We show empirically that models confidently generate multiple PD categories for well-known individuals. For everyday users, GPT-4o generates 11 features with 60% or more accuracy (e.g., gender, hair color, languages). Finally, 72% of participants sought control over model-generated associations with their name, raising questions about what counts as PD and whether data privacy rights should extend to LLMs.

*   **研究动机**：LLM 在海量网页抓取数据上训练，不可避免地接触到个人数据（PD）。虽然已知模型会“背诵”训练数据，但普通用户无法得知模型究竟将哪些具体信息与其姓名关联。
*   **核心问题**：流行的 LLM（尤其是闭源 API 模型）在多大程度上能准确关联普通人的个人特征？用户对这些关联的感知如何？现有的隐私权利（如 GDPR 的“被遗忘权”）是否应延伸至模型生成的关联信息？
*   **整体含义**：研究填补了技术性隐私审计与用户心理感知之间的空白，开发了首个面向普通用户的 LLM 隐私自检工具。

### 2. 方法论
*   **核心思想**：将针对开源模型的隐私探测框架（WikiMem）适配为适用于闭源 API 的黑盒审计方法，并开发了名为 **LMP2**（Language Model Privacy Probe）的浏览器工具。
*   **关键技术细节**：
    *   **双字符前缀探测（Two-character prefixing）**：由于无法直接获取闭源 API 的全量 Token 概率，研究者向模型发送带有“前缀”的补全指令（例如：“[姓名]的出生日期是 19...”），观察模型补全结果。
    *   **指令式句子校正**：使用特定的系统提示词（System Prompt）要求模型仅输出补全后的单词，以标准化不同模型家族的输出。
    *   **关联强度与置信度计算**：
        *   **关联强度**：结合模型输出特定值的频率和平均概率（或投票权重）。
        *   **置信度**：通过分布偏度（Skewness）和主导比例（Dominance Ratio）衡量模型输出是否收敛于单一结果。
    *   **基准校准（Baseline Calibration）**：引入“通用主体”（如“这个人的...”）作为基准，减去模型对特定属性的先验偏好（如默认所有人都说英语），从而提取出与特定姓名相关的关联。

### 3. 实验设计
*   **审计模型**：共 8 款模型。包括开源模型（Qwen3 4B, Llama 3.1 8B, Ministral 8B）和 API 模型（GPT-4o, Gemini Flash 2.0, GPT-5, Grok-3, Command R）。
*   **数据集与场景**：
    *   **Famous（知名人士）**：100 位在维基数据中有详尽记录的公众人物，用于测试模型在理想条件下的记忆能力。
    *   **Synthetic（合成姓名）**：100 个不存在的姓名，作为推理能力的基准（排除记忆干扰）。

*   **对比维度**：对比了不同规模模型、不同敏感度属性（共 50 类特征，如性别、宗教、电话、发色等）的准确率和置信度。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 8 种模型、50 种人类属性、每种属性 5 种同义改写（Canaries）。
*   **充分性**：实验设计非常全面。不仅有大规模的自动化模型审计，还包含了形成性研究（N=20）来优化工具 UI，以及针对数百名真实用户的实证研究。通过引入“合成姓名”和“通用主体基准”，有效地隔离了“记忆”与“统计推理”的影响，实验逻辑客观、公平。

### 6. 主要结论与发现
*   **高准确性关联**：对于普通用户，GPT-4o 能以超过 60% 的准确率预测 11 项特征（如性别、发色、母语、性取向等）。
*   **知名人士隐私泄露**：大型 API 模型能以极高精度（Precision > 0.8）复现公众人物的敏感信息（如宗教、政党、出生日期）。
*   **置信度与准确度脱节**：模型有时会以极高置信度输出错误信息（如默认所有人的电话前缀都是 +1），表现出系统性偏差。
*   **用户诉求强烈**：72% 的参与者在看到审计结果后，希望拥有删除或更正模型生成的与其姓名相关的关联信息的权利（即行使被遗忘权）。

### 7. 优点
*   **以人为中心**：将深奥的隐私审计转化为普通用户可操作、可理解的工具。
*   **黑盒适配性**：提出了一套在不接触模型内部参数的情况下，评估闭源 API 隐私风险的有效流程。
*   **数据最小化**：工具设计遵循隐私保护原则，仅向服务器发送前缀，不存储用户的完整真实信息。

### 8. 不足与局限
*   **样本偏差**：参与者主要来自 Prolific 平台，年轻人和男性比例偏高，且集中在部分欧盟国家（如波兰、葡萄牙）。
*   **特征限制**：审计仅限于预定义的 50 类特征，可能无法覆盖所有潜在的隐私风险。
*   **推理与记忆的界限**：虽然努力区分，但在某些情况下（如根据姓名推测国籍），仍难以完全剥离模型是基于“记忆”还是基于“强大的统计推理”。
*   **单 Token 偏向**：前缀补全方法更倾向于生成短答案，对于长描述性属性的审计效果有限。
