## 摘要
个性化 AI 助手必须能够召回并推理长期用户记忆，这些记忆自然地跨越了多种模态和来源，如图像、视频和电子邮件。然而，现有的长期记忆基准主要关注对话历史，未能捕捉到基于生活经验的真实个性化指代。我们推出了 ATM-Bench，这是首个针对多模态、多源个性化指代记忆问答的基准。ATM-Bench 包含约四年的隐私保护个人记忆数据，以及带有真实记忆证据的人工标注问答对，其中包括需要解析个人指代、多源多证据推理以及处理冲突证据的查询。我们提出了模式引导记忆（Schema-Guided Memory, SGM），用于结构化地表示源自不同渠道的记忆项。在实验中，我们实现了 5 种最先进的记忆系统以及一个标准的 RAG 基准，并评估了具有不同记忆摄取、检索和答案生成技术的变体。我们发现 ATM-Bench-Hard 集上的性能较差（准确率低于 20%），并且 SGM 相比于先前工作中常用的描述性记忆（Descriptive Memory）提升了性能。代码可在以下网址获取：https://github.com/JingbiaoMei/ATM-Bench

## Abstract
Personalized AI assistants must recall and reason over long-term user memory, which naturally spans multiple modalities and sources such as images, videos, and emails. However, existing Long-term Memory benchmarks focus primarily on dialogue history, failing to capture realistic personalized references grounded in lived experience. We introduce ATM-Bench, the first benchmark for multimodal, multi-source personalized referential Memory QA. ATM-Bench contains approximately four years of privacy-preserving personal memory data and human-annotated question-answer pairs with ground-truth memory evidence, including queries that require resolving personal references, multi-evidence reasoning from multi-source and handling conflicting evidence. We propose Schema-Guided Memory (SGM) to structurally represent memory items originated from different sources. In experiments, we implement 5 state-of-the-art memory systems along with a standard RAG baseline and evaluate variants with different memory ingestion, retrieval, and answer generation techniques. We find poor performance (under 20\% accuracy) on the ATM-Bench-Hard set, and that SGM improves performance over Descriptive Memory commonly adopted in prior works. ### 2. 方法论
论文提出了 **ATM-Bench** 基准和 **Schema-Guided Memory (SGM)** 框架：
*   **ATM-Bench 构建**：收集了约 4 年的真实个人数据，经过严格的隐私脱敏（人脸模糊、邮件重写、PII 替换），由人工标注了 1038 个问答对及对应的证据链。
*   **SGM (模式引导记忆)**：
    *   **核心思想**：将异构的原始数据（图像、视频、邮件）转化为统一的结构化键值对（Key-Value Fields）。
    *   **关键字段**：包括时间（Time）、地点（Location）、实体（Entities）、OCR 文本、标签（Tags）等。
    *   **对比对象**：传统的描述性记忆（Descriptive Memory, DM），即直接将记忆转化为一段自然语言描述。
*   **系统流程**：
    1.  **记忆摄取（Ingestion）**：预处理原始数据并组织成“堆叠式（Piled）”或“链接式（Linked，图结构）”。
    2.  **检索（Retrieval）**：基于向量嵌入（Embedding）进行相似度搜索。
    3.  **答案生成（QA）**：单次生成或通过 Agent 进行迭代推理。

### 6. 主要结论与发现
*   **性能鸿沟**：在 ATM-Bench-Hard 集合上，所有现有系统的准确率均**低于 20%**，说明长期复杂记忆推理仍是极大的挑战。
*   **SGM 的优越性**：结构化的 SGM 在检索和 QA 性能上均显著优于非结构化的 DM（描述性记忆）。
*   **Agent 策略的局限**：在简单任务上 Agent 迭代有帮助，但在极困难任务上，目前的 Agent 推理策略并未表现出明显优势。
*   **推理上限**：即使在 Oracle（提供完美证据）的情况下，最强模型（GPT-5）在困难集上的得分也仅为 74.7，说明多证据聚合推理本身就很难。

### 7. 优点
*   **真实性与隐私平衡**：使用了长达 4 年的真实跨国生活数据，同时设计了极其严密的隐私保护流程。
*   **高质量标注**：通过人工标注捕捉到了 LLM 难以生成的“人类式”模糊查询。
*   **方法实用**：提出的 SGM 框架简单有效，易于在端侧设备（如手机助手）上部署。

### 8. 不足与局限
*   **数据规模**：虽然时间跨度长，但总问答对数量（1000+）相对较小，可能不足以支撑大规模模型的微调。
*   **单用户偏差**：数据主要源于少数贡献者，可能无法完全覆盖全球所有文化背景下的记忆习惯。
*   **视频处理简化**：为了节省上下文，视频仅抽取了少量帧（最多 8 帧），可能丢失了部分动态信息。
