## 摘要
由于巨大的计算需求和隐私担忧，在实时系统中部署大语言模型（LLMs）仍然具有挑战性。我们提出了 Floe，这是一个专为延迟敏感、资源受限环境设计的混合联邦学习框架。Floe 将云端黑盒 LLM 与边缘设备上的轻量级小语言模型（SLMs）相结合，以实现低延迟、保护隐私的推理。个人数据和微调保留在设备端，而云端 LLM 在不暴露专有权重的情况下贡献通用知识。一种异构感知的 LoRA 适配策略实现了跨多样化硬件的高效边缘部署，而 Logit 级融合机制实现了边缘模型与云端模型之间的实时协同。大量实验表明，Floe 增强了用户隐私和个性化。此外，与基准方法相比，它在实时约束下显著提高了模型性能并降低了边缘设备上的推理延迟。

## Abstract
Deploying large language models (LLMs) in real-time systems remains challenging due to their substantial computational demands and privacy concerns. We propose Floe, a hybrid federated learning framework designed for latency-sensitive, resource-constrained environments. Floe combines a cloud-based black-box LLM with lightweight small language models (SLMs) on edge devices to enable low-latency, privacy-preserving inference. Personal data and fine-tuning remain on-device, while the cloud LLM contributes general knowledge without exposing proprietary weights. A heterogeneity-aware LoRA adaptation strategy enables efficient edge deployment across diverse hardware, and a logit-level fusion mechanism enables real-time coordination between edge and cloud models. Extensive experiments demonstrate that Floe enhances user privacy and personalization. Moreover, it significantly improves model performance and reduces inference latency on edge devices under real-time constraints compared with baseline approaches.

### 1. 核心问题与整体含义（研究动机和背景）
在实时系统中部署 LLM 面临三大矛盾：
*   **隐私与性能的冲突：** 云端 LLM（如 GPT-4）性能强劲但存在隐私泄露风险；边缘端小模型（SLM）保护隐私但推理能力有限。
*   **硬件限制与模型规模的冲突：** 边缘设备（如 Jetson Nano）内存极小，无法运行百亿级参数的模型，甚至无法进行全参数微调。
*   **知识产权保护：** 顶尖模型多为闭源黑盒，无法获取其内部参数进行传统的联邦学习。
**Floe 的目标：** 通过云端黑盒 LLM 与边缘端个性化 SLM 的协同，在保护隐私和模型 IP 的前提下，实现高性能、低延迟的实时推理。

### 2. 方法论：核心思想与关键技术
Floe 框架分为两个阶段：**联邦微调阶段**和**协同推理阶段**。

*   **联邦微调阶段（Fine-tuning Phase）：**
    *   **异构感知 LoRA 训练器：** 针对边缘设备不同的内存和算力，动态调整 LoRA 的秩（Rank）。内存大、延迟容忍度高的设备使用高秩，反之使用低秩，以确保所有设备都能参与训练。
    *   **任务特定聚合器：** 服务器根据 LoRA 模块的语义相似性进行聚类（Clustering），而不是简单的全局平均。这解决了数据非独立同分布（Non-IID）问题，使模型能针对特定任务（如医疗、代码）进行专业化。
*   **协同推理阶段（Inference Phase）：**
    *   **隐私检测器（Privacy Detector）：** 在本地预先判断输入是否包含敏感信息。敏感请求强制留在本地 SLM 处理，非敏感请求则触发云边协同。
    *   **参数无关的 MoE 路由：** 在边缘端动态合并多个专家 LoRA 模块，无需额外训练门控网络，通过计算 Prompt 与专家语义的相似度来分配权重。
    *   **Logit 级对齐融合：** 这是核心技术。云端 LLM 输出通用知识的 Logit 分布，边缘端 SLM 输出带有本地上下文的 Logit 分布。通过一个轻量级对齐模块，将两者融合生成最终 Token，既保留了 LLM 的推理能力，又融入了 SLM 的个性化信息。

### 6. 主要结论与发现
*   **性能提升：** Floe 在 BBH 基准上比纯 SLM 提升了 14.2% 的准确率，比纯云端 LLM 提升了 3.6%。
*   **系统效率：** 相比传统模型压缩方案，Floe 减少了 99% 的通信延迟，显著降低了边缘端的内存占用和能耗。
*   **实时鲁棒性：** 引入了超时回退机制，当网络延迟超过 200ms 时，系统自动切换为纯本地 SLM 推理，确保了用户体验的流畅性。
*   **隐私保护：** 隐私检测器达到了 94.3% 的 F1 分数，有效阻止了敏感数据上传。

### 7. 优点与亮点
*   **黑盒兼容性：** 创新的 Logit 级融合不需要访问云端模型的权重，非常适合当前闭源 API 主导的生态。
*   **异构适应性：** 动态 LoRA 秩选择解决了联邦学习中“木桶效应”（慢掉队者影响全局）的问题。
*   **端到端闭环：** 方案不仅关注模型精度，还深入到了隐私检测、网络波动处理等实际部署痛点。

### 8. 不足与局限
*   **隐私检测风险：** 隐私检测器基于启发式规则和语义相似度，存在约 8.3% 的漏检率（Recall 为 91.7%），极端敏感场景下仍有泄露风险。
*   **词表对齐假设：** Logit 融合假设云端和边缘模型使用可映射的词表（Tokenizer）。如果两者词表完全不兼容，该方法将失效。
*   **硬件性能与数据价值的错配：** 异构策略会给弱硬件分配低秩，如果该设备拥有高质量数据，其贡献可能会因模型容量受限而无法充分发挥。
