## 摘要
大语言模型（LLMs）正越来越多地充当个人助手，用户在长期的交互过程中会分享复杂且多样的偏好。然而，评估 LLMs 在真实的长期情境下遵循这些偏好的能力仍有待深入探索。本研究提出了 RealPref，这是一个用于评估个性化用户-LLM 交互中真实偏好遵循能力的基准测试。RealPref 包含 100 个用户画像、1300 个个性化偏好、四种偏好表达类型（从显式到隐式）以及长程交互历史。它包含三种类型的测试题（多选题、判断题和开放式问题），并为 LLM-as-a-judge 评估提供了详细的评分标准。结果表明，随着上下文长度的增加和偏好表达变得更加隐式，LLM 的性能显著下降，且将用户偏好理解泛化到未见场景也带来了进一步的挑战。RealPref 及其研究结果为未来开发能够更好适应个人需求的用户感知型 LLM 助手奠定了基础。代码可在 https://github.com/GG14127/RealPref 获取。

## Abstract
Large Language Models (LLMs) are increasingly serving as personal assistants, where users share complex and diverse preferences over extended interactions. However, assessing how well LLMs can follow these preferences in realistic, long-term situations remains underexplored. This work proposes RealPref, a benchmark for evaluating realistic preference-following in personalized user-LLM interactions. RealPref features 100 user profiles, 1300 personalized preferences, four types of preference expression (ranging from explicit to implicit), and long-horizon interaction histories. It includes three types of test questions (multiple-choice, true-or-false, and open-ended), with detailed rubrics for LLM-as-a-judge evaluation. Results indicate that LLM performance significantly drops as context length grows and preference expression becomes more implicit, and that generalizing user preference understanding to unseen scenarios poses further challenges. RealPref and these findings provide a foundation for future research to develop user-aware LLM assistants that better adapt to individual needs. The code is available at https://github.com/GG14127/RealPref.

随着大语言模型（LLM）在个人助手领域的广泛应用，用户期望模型能通过长期交互记住并遵循其复杂的个性化偏好。然而，现有的个性化评估基准存在三大缺陷：
*   **上下文过于简单**：多关注短程对话或单一指令，而非真实的长期交互。
*   **偏好表达过于直接**：假设偏好总是显式给出的，忽略了现实中隐式、零散的表达方式。
*   **评估方法局限**：多采用分类任务，与自然交互脱节，且缺乏对生成质量的细粒度评估。
本文提出了 **RealPref** 基准，旨在评估 LLM 在长程交互中捕捉、保留、遵循及泛化用户偏好的能力。

#### 2. 论文提出的方法论
RealPref 采用多阶段生成管线（基于 GPT-4o）构建了一个模拟真实交互的数据集：
*   **用户画像构建**：从 PersonaHub 采样 100 个种子画像，扩展为包含人口统计学信息、详细传记和生命周期事件的完整 Profile。
*   **偏好生成**：为每个用户生成 10 个原始偏好（涵盖 37 个子话题）和 3 个可推导的泛化偏好。要求偏好具备多样性、独特性且非普适性（Value-free）。
*   **四种偏好表达类型**：
    1.  **显式-直接陈述**：直接告知偏好。
    2.  **显式-上下文提及**：在多轮对话中自然提及。
    3.  **隐式-风格表达**：通过修辞、隐喻或情感暗示偏好。
    4.  **隐式-经验反馈**：偏好分散在多个 Session 中，需整合反馈信息进行推理。
*   **长程上下文构造**：通过插入用户无关的随机对话，将上下文长度从 2K 扩展至最高 247K tokens。
*   **评估框架**：包含多选题（MCQ）、判断题（T/F）和开放式生成题。针对生成题，提出基于 **LLM-as-a-judge** 的三维评分标准（1-5分）：偏好感知度、偏好对齐度、回答质量。

#### 6. 论文的主要结论与发现
*   **性能随复杂度下降**：随着偏好表达从显式变为隐式，所有模型的性能均显著下降。
*   **长上下文挑战**：模型性能随上下文长度增加而降低，且“偏好信息与查询之间的距离”比总长度对性能的影响更大。
*   **泛化能力缺失**：LLM 在处理需要从已知偏好推导至新场景的任务时表现较差，缺乏主动泛化的意识。
*   **改进手段有效性**：在模型处理能力范围内，简单的 Reminder 效果显著；在超长上下文中，RAG 是提升性能最有效的手段。
*   **题型敏感性**：MCQ 容易被模型通过选项对比“破解”，而 T/F 和开放式生成更能真实反映模型的偏好遵循能力。

#### 7. 优点
*   **真实性高**：模拟了用户逐渐透露信息的过程，特别是“隐式经验反馈”类型非常贴近现实。
*   **细粒度评估**：不再局限于二元对错，而是从感知、对齐、质量三个维度进行 1-5 分的深度评价。
*   **泛化测试设计**：首次系统性地测试了模型是否能“举一反三”理解用户的底层价值观。

#### 8. 不足与局限
*   **合成数据偏差**：所有数据均为合成，可能无法完全捕捉人类行为的不可预测性或文化细微差别。
*   **静态偏好**：实验假设用户偏好是静态的，未考虑现实中偏好可能随时间演变或存在冲突的情况。
*   **评估成本**：依赖高级 LLM 作为评委，在大规模实时评估中成本较高。
*   **多模态缺失**：目前仅限于文本交互，未涉及语音、图像等更丰富的个性化信号。
