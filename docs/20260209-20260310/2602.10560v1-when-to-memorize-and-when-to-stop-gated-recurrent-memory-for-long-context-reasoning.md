## 摘要
尽管长上下文推理对各种现实应用至关重要，但对于大语言模型（LLMs）而言仍具挑战性，因为其性能会随上下文长度的增加而下降。近期工作 MemAgent 尝试通过在类 RNN 循环中逐块处理上下文并更新文本记忆以进行最终回答来解决此问题。然而，这种朴素的循环记忆更新存在两个关键缺陷：(i) 记忆可能因不加区分地更新（即使是在无证据的块上）而迅速膨胀；(ii) 循环缺乏退出机制，导致在收集到充足证据后仍产生不必要的计算。为解决这些问题，我们提出了 GRU-Mem，它结合了两个文本控制门，以实现更稳定、更高效的长上下文推理。具体而言，在 GRU-Mem 中，记忆仅在更新门开启时更新，且一旦退出门开启，循环将立即终止。为了使模型具备这些能力，我们在端到端强化学习（RL）中引入了两个奖励信号 $r^{\text{update}}$ 和 $r^{\text{exit}}$，分别对正确的更新和退出行为进行奖励。在各种长上下文推理任务上的实验证明了 GRU-Mem 的有效性和效率，其性能普遍优于原始 MemAgent，且推理速度提升高达 400%。

## Abstract
While reasoning over long context is crucial for various real-world applications, it remains challenging for large language models (LLMs) as they suffer from performance degradation as the context length grows. Recent work MemAgent has tried to tackle this by processing context chunk-by-chunk in an RNN-like loop and updating a textual memory for final answering. However, this naive recurrent memory update faces two crucial drawbacks: (i) memory can quickly explode because it can update indiscriminately, even on evidence-free chunks; and (ii) the loop lacks an exit mechanism, leading to unnecessary computation after even sufficient evidence is collected. To address these issues, we propose GRU-Mem, which incorporates two text-controlled gates for more stable and efficient long-context reasoning. Specifically, in GRU-Mem, the memory only updates when the update gate is open and the recurrent loop will exit immediately once the exit gate is open. To endow the model with such capabilities, we introduce two reward signals $r^{\text{update}}$ and $r^{\text{exit}}$ within end-to-end RL, rewarding the correct updating and exiting behaviors respectively. Experiments on various long-context reasoning tasks demonstrate the effectiveness and efficiency of GRU-Mem, which generally outperforms the vanilla MemAgent with up to 400\% times inference speed acceleration.

*   **背景**：前作 MemAgent 提出了一种类 RNN 的循环记忆范式，将长文本分块处理并维护一个文本记忆。但 MemAgent 存在两个致命缺陷：
    1.  **记忆爆炸**：模型会不加区分地更新记忆，导致无关信息堆积，最终超出长度限制。
    2.  **缺乏退出机制**：即使已经找到了足以回答问题的证据，模型仍必须处理完所有文本块，造成巨大的计算浪费。
*   **核心目标**：引入门控机制，让模型学会“何时记忆”以及“何时停止”。

### 2. 方法论
*   **核心思想**：借鉴经典循环神经网络 GRU 的设计，提出 **GRU-Mem**。在循环处理流程中引入两个由文本控制的逻辑门：**更新门（Update Gate, UG）**和**退出门（Exit Gate, EG）**。
*   **关键技术细节**：
    *   **工作流**：模型在处理每个文本块时，需输出：`<think>`（推理过程）、`<check>`（更新门状态：yes/no）、`<update>`（候选记忆内容）和 `<next>`（退出门状态：continue/end）。
    *   **更新逻辑**：只有当 UG 为 "yes" 时，才用新生成的候选记忆替换旧记忆；否则保持旧记忆不变。
    *   **退出逻辑**：一旦 EG 为 "end"，立即终止循环，直接进入回答阶段。
*   **强化学习优化**：
    *   使用 **Multi-Conv DAPO**（基于 GRPO 的多轮对话优化算法）进行端到端训练。
    *   **奖励设计**：除了最终答案正确性的“结果奖励”外，引入了 **$r^{update}$**（奖励在含证据块时更新、无证据块时不更新的行为）和 **$r^{exit}$**（奖励在最后一个证据出现时准确退出的行为）。
    *   **优势计算**：通过超参数 $\alpha$ 平衡轨迹级（Trajectory-level）和回合级（Turn-level）的优势函数，引导模型学习复杂的门控逻辑。

*   **记忆稳定性**：更新门有效抑制了无关信息的累积，成功解决了记忆爆炸问题，使记忆长度保持在健康水平。
*   **RL 的关键性**：消融实验证明，如果没有针对性的 RL 奖励，模型很难自发学会精准的门控操作。

### 7. 优点
*   **机制创新**：将经典的 GRU 门控思想成功移植到 LLM 的文本记忆管理中，逻辑清晰且直观。

*   **灵活性**：推理时可根据任务需求选择是否开启退出门（w/o EG 模式仍比原版快，因为记忆更短）。

### 8. 不足与局限
*   **任务覆盖面**：目前主要集中在问答（QA）领域，对于长文本摘要、创意写作等需要全局感知的任务尚未验证。

*   **收敛速度**：由于奖励函数变得复杂，模型可能需要更长的收敛时间。
