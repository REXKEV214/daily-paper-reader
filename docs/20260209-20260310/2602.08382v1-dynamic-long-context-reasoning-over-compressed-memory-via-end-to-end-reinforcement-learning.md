## 摘要
大语言模型（LLMs）在长上下文处理中面临重大挑战，包括二次方计算开销、信息遗忘以及检索增强生成（RAG）中固有的上下文碎片化问题。我们提出了一种受认知启发的框架，用于高效的长上下文推理，该框架基于分块压缩和选择性记忆召回，而非处理所有原始 token。该框架将长输入分割成块，并使用学习到的压缩器将每个块编码为压缩记忆表示。门控模块动态选择相关的记忆块，随后由推理模块利用不断演进的工作记忆进行迭代处理，以解决下游任务。压缩器和推理器通过端到端强化学习进行联合优化，而门控模块则作为分类器进行单独训练。实验结果表明，所提方法在 RULER-HQA 等多跳推理基准测试中达到了具有竞争力的准确率，将上下文长度从 7K 外推至 1.75M token，并且与强大的长上下文基准模型相比，提供了良好的准确率-效率权衡。特别是，与 MemAgent 相比，它实现了高达 2 倍的 GPU 显存峰值占用降低和 6 倍的推理加速。

## Abstract
Large Language Models (LLMs) face significant challenges in long-context processing, including quadratic computational costs, information forgetting, and the context fragmentation inherent in retrieval-augmented generation (RAG). We propose a cognitively inspired framework for efficient long-context inference based on chunk-wise compression and selective memory recall, rather than processing all raw tokens. The framework segments long inputs into chunks and encodes each chunk into compressed memory representations using a learned compressor. A gating module dynamically selects relevant memory blocks, which are then iteratively processed by a reasoning module with an evolving working memory to solve downstream tasks. The compressor and reasoner are jointly optimized via end-to-end reinforcement learning, while the gating module is trained separately as a classifier. Experimental results show that the proposed method achieves competitive accuracy on multi-hop reasoning benchmarks such as RULER-HQA, extrapolates context length from 7K to 1.75M tokens, and offers a favorable accuracy-efficiency trade-off compared to strong long-context baselines. In particular, it achieves up to a 2 times reduction in peak GPU memory usage and a 6 times inference speedup over MemAgent.

*   **上下文碎片化**：检索增强生成（RAG）将文本切碎，破坏了多跳推理所需的逻辑依赖。

本文提出的 **LycheeMemory** 旨在通过模拟人类的“长期压缩记忆”与“动态工作记忆”机制，实现百万级长度上下文的高效、精准推理。

## 2. 方法论
LycheeMemory 框架由三个核心组件构成，通过三阶段流程运行：

*   **核心组件**：
    1.  **Compressor (压缩器 Φcomp)**：利用 LoRA 增强的基座模型，将原始文本块（如 4096 tokens）编码为少量的“记忆 Token”（KV-cache 风格的隐藏状态）。
    2.  **Gate (门控 Φgate)**：一个轻量级分类器，根据当前查询（Query）和不断演进的“工作记忆”状态，动态决定当前压缩块是否与任务相关。
    3.  **Reasoner (推理器 Φreason)**：负责处理被选中的记忆块，更新工作记忆，并最终生成答案。

*   **关键技术细节与流程**：
    1.  **分块压缩**：将长文档切分为固定大小的块，通过压缩器生成高维潜在表示，构建压缩记忆库。
    2.  **动态召回与推理**：模型顺序扫描压缩记忆库。Gate 模块进行过滤，只有通过阈值的块才会触发 Reasoner 进行工作记忆更新。
    3.  **端到端强化学习 (RL)**：这是本文的核心创新。研究者使用 **GSPO (Group Sequence Policy Optimization)** 算法，将压缩器和推理器进行联合优化。这意味着压缩过程不再是静态的，而是为了最大化最终推理的准确性而学习“如何压缩”。

## 7. 优点
*   **端到端优化思想**：通过 RL 让压缩器学会为推理而压缩，解决了隐式表示与显式推理之间的对齐问题。
*   **推理成本可控**：由于 Gate 的高效过滤，推理时间随文本长度增长的斜率极低，接近常数级开销。
*   **认知架构合理**：区分长期和短期记忆，既保留了原始信息的语义，又避免了全量计算的冗余。

## 8. 不足与局限
*   **单向扫描限制**：模型采用顺序扫描，无法回溯。如果 Gate 在早期误判并丢弃了关键信息，后续推理将无法补救（占比约 35% 的错误原因）。
*   **特征坍缩风险**：在极高压缩比下，相似的实体（如双胞胎的名字或接近的日期）可能在潜在空间中混淆，导致细微的幻觉。
*   **训练门槛**：三阶段训练流程（压缩预训练 -> 联合 RL -> Gate 训练）相对复杂，对奖励函数的设计和数据质量要求较高。
