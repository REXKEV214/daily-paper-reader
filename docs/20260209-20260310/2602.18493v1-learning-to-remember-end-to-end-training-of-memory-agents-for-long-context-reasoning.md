## 摘要
长上下文大语言模型（LLM）和检索增强生成（RAG）系统被动地处理信息，将状态追踪、矛盾解决和证据聚合推迟到查询阶段，这在包含频繁更新的超长流式数据下会变得脆弱。我们提出了统一记忆智能体（Unified Memory Agent, UMA），这是一个端到端的强化学习框架，在单一策略中统一了记忆操作和问答。UMA 维护一种双重记忆表示：用于全局上下文的紧凑核心摘要，以及支持对键值条目进行显式 CRUD（创建、更新、删除、重组）操作的结构化记忆库（Memory Bank），从而在流式处理过程中实现主动整合。为了评估长程记忆行为，我们引入了 Ledger-QA，这是一个用于连续状态追踪的诊断基准，其中的答案是源自累积更新的潜在值，而非局部的片段检索。在涵盖 Ledger-QA、测试时学习（Test-Time Learning）和准确检索的 13 个数据集上，UMA 在动态推理和学习任务上显著优于长上下文和 RAG 基线，同时在标准检索基准上保持了竞争力，强调了学习型端到端记忆管理的重要性。

## Abstract
Long-context LLMs and Retrieval-Augmented Generation (RAG) systems process information passively, deferring state tracking, contradiction resolution, and evidence aggregation to query time, which becomes brittle under ultra long streams with frequent updates. We propose the Unified Memory Agent (UMA), an end-to-end reinforcement learning framework that unifies memory operations and question answering within a single policy. UMA maintains a dual memory representation: a compact core summary for global context and a structured Memory Bank that supports explicit CRUD (create, update, delete, reorganize) over key value entries, enabling proactive consolidation during streaming. To evaluate long-horizon memory behavior, we introduce Ledger-QA, a diagnostic benchmark for continuous state tracking where answers are latent values derived from accumulated updates rather than lo cal span retrieval. Across 13 datasets spanning Ledger-QA, Test-Time Learning, and Accurate Retrieval, UMA substantially outperforms long-context and RAG baselines on dynamic reasoning and learning tasks while remaining competitive on standard retrieval benchmarks, underscoring the importance of learned, end-to-end memory management.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：现有的长上下文大语言模型（LLM）和检索增强生成（RAG）系统在处理超长信息流时表现出“被动性”。它们通常将状态追踪、矛盾解决和证据聚合推迟到查询阶段，导致在面对频繁更新的动态信息（如财务流水、长期对话）时，推理效率低下且容易出现“上下文腐烂”（Context Rot）现象。
*   **研究动机**：作者认为智能体需要具备“主动记忆”能力，即在信息到达时就进行整合与提炼，而不是简单地堆叠原始文本。现有的记忆智能体多依赖手工规则或解耦训练，无法实现记忆管理与任务执行的协同优化。

### 2. 方法论：核心思想与关键技术
*   **核心思想**：提出**统一记忆智能体（Unified Memory Agent, UMA）**，通过端到端强化学习（RL），将记忆的增、删、改、查（CRUD）操作与最终的问答任务统一在同一个策略网络中。
*   **关键技术细节**：
    *   **双重记忆表示**：包含一个**核心摘要（Core Summary）**用于维护全局上下文，以及一个**结构化记忆库（Memory Bank）**用于存储精确的键值对（Key-Value）条目。
    *   **两阶段工作流**：
        1.  **记忆维护阶段**：逐块处理输入，智能体自主决定是否创建、更新或删除记忆条目。
        2.  **混合检索问答阶段**：结合结构化记忆库检索和原始文本的传统检索（BM25/Embedding）来回答问题。
    *   **任务分层 GRPO 算法（Task-Stratified GRPO）**：
        *   针对记忆操作和问答任务的不同性质，改进了 GRPO 强化学习算法。
        *   **嵌套采样**：对同一上下文采样多个记忆轨迹，再对每个记忆状态采样多个问题。
        *   **分层优势估计**：记忆操作的奖励取决于其对未来所有潜在问题的平均贡献（未来效用信号），而问答操作的奖励则在同问题的不同轨迹间进行归一化，确保信用分配的公平性。

### 3. 实验设计
*   **数据集与场景**：涵盖 13 个数据集，分为三大类：
    1.  **动态状态追踪**：新提出的 **Ledger-QA**（模拟长达一年的个人财务管理，需处理 2 到 50 个会话的动态更新）。
    2.  **测试时学习（TTL）**：如 Banking77、Clinic、NLU 等，要求智能体在上下文中学习分类规则。
    3.  **准确检索（AR）**：如 HotpotQA、SQuAD、MSC 等标准长文本检索任务。
*   **对比方法**：包括全文本拼接（Concat）、标准 RAG、循环摘要模型（MemAgent）、以及最先进的训练型记忆智能体（Mem1, MemAlpha）。

### 4. 资源与算力
*   **算力投入**：使用了由 **32 台 NVIDIA H200 GPU**（4 个节点，每节点 8 卡）组成的集群。
*   **训练时长**：通用智能体（Generalist Agent）训练约 **2 天**；针对 Ledger-QA 的专用智能体（Specialist Agent）训练约 **0.5 天**。
*   **模型底座**：策略网络初始化自 `Qwen3-4B-Instruct`。

### 5. 实验数量与充分性
*   **实验规模**：在 13 个不同性质的数据集上进行了广泛测试，涵盖了从数千到数十万 token 的上下文长度。
*   **消融实验**：设计了多组对比，包括“无强化学习”、“无记忆维护阶段”、“全局归一化 vs 分层归一化”以及“两阶段解耦训练 vs 端到端训练”。
*   **客观性**：通过 Ledger-QA 模拟了不同长度（2-50 session）的压力测试，并使用 `Qwen3-30B` 作为公正的第三方评测器（LLM-as-a-Judge），实验设计较为严谨、客观。

### 6. 主要结论与发现
*   **性能领先**：UMA 在动态推理和状态追踪任务上显著优于所有基线。在 Ledger-QA 的 50 会话极长测试中，UMA 保持了 >50% 的准确率，而其他基线均跌至 25% 以下。
*   **端到端的重要性**：消融实验证明，端到端联合优化优于两阶段解耦训练，因为前者能让记忆操作提前感知并适应未来的查询需求。
*   **泛化能力**：尽管在特定任务上训练，UMA 在未见过的 TTL 数据集（如 Banking77）上依然表现出极强的泛化性能。

### 7. 优点与亮点
*   **主动性**：将记忆管理从“被动检索”转变为“主动维护”，更符合人类处理长期信息的逻辑。
*   **算法创新**：提出的 Task-Stratified GRPO 有效解决了长程记忆任务中奖励稀疏和信用分配难的问题。
*   **基准贡献**：Ledger-QA 填补了现有长文本评测中缺乏“动态状态聚合”评估的空白。

### 8. 不足与局限
*   **硬性窗口限制**：虽然有记忆库，但单次处理仍受限于 16k 的硬性上下文窗口截断。
*   **合成数据偏差**：Ledger-QA 虽然严谨但属于合成数据，真实世界的财务或状态追踪可能包含更多非结构化噪声和歧义。
*   **模态限制**：目前仅支持文本，尚未扩展到多模态记忆（如图像、音频流）的管理。
