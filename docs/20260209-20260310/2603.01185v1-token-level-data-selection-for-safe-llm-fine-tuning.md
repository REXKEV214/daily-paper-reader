## 摘要
在自定义数据集上微调大语言模型（LLMs）已成为将这些模型适配到特定领域和应用的标准方法。然而，最近的研究表明，此类微调可能导致模型安全性显著下降。现有的防御方法在样本级别运行，且往往在安全性和实用性之间难以达到令人满意的权衡。为了解决这一局限性，我们对微调过程中的安全性下降进行了系统的 Token 级诊断。基于此，我们提出了面向安全大语言模型微调的 Token 级数据选择（TOSS），这是一个通过测量安全性下降模型与实用导向模型之间的损失差异来量化每个 Token 安全风险的新型框架。这种 Token 级的粒度能够准确识别并移除不安全的 Token，从而保留有价值的任务特定信息。此外，我们引入了一种渐进式细化策略 TOSS-Pro，该策略迭代地增强安全性下降模型识别不安全 Token 的能力。大量实验表明，我们的方法在微调过程中稳健地保护了大语言模型，同时实现了卓越的下游任务性能，显著优于现有的样本级防御方法。我们的代码可在 https://github.com/Polly-LYP/TOSS 获取。

## Abstract
Fine-tuning large language models (LLMs) on custom datasets has become a standard approach for adapting these models to specific domains and applications. However, recent studies have shown that such fine-tuning can lead to significant degradation in the model's safety. Existing defense methods operate at the sample level and often suffer from an unsatisfactory trade-off between safety and utility. To address this limitation, we perform a systematic token-level diagnosis of safety degradation during fine-tuning. Based on this, we propose token-level data selection for safe LLM fine-tuning (TOSS), a novel framework that quantifies the safety risk of each token by measuring the loss difference between a safety-degraded model and a utility-oriented model. This token-level granularity enables accurate identification and removal of unsafe tokens, thereby preserving valuable task-specific information. In addition, we introduce a progressive refinement strategy, TOSS-Pro, which iteratively enhances the safety-degraded model's ability to identify unsafe tokens. Extensive experiments demonstrate that our approach robustly safeguards LLMs during fine-tuning while achieving superior downstream task performance, significantly outperforming existing sample-level defense methods. Our code is available at https://github.com/Polly-LYP/TOSS.

*   **研究背景**：微调是适配 LLM 到特定领域的标准做法，但研究发现，即使是看似无害的自定义数据，也可能破坏模型预先对齐的安全护栏，导致模型产生有害输出。
*   **核心痛点**：现有的防御策略（如数据混合或样本级过滤）粒度太粗。样本级过滤（如 SEAL）会直接丢弃整个样本，这不仅损失了样本中包含的有用任务信息，还忽略了“有害信号”往往只集中在特定 Token 上的事实。
*   **核心假设**：安全性退化的基本单位不是样本，而是 **Token（令牌）**。论文通过诊断发现，有害信号主要集中在响应的初始 Token 以及散布在中间的特定词汇中。

### 2. 方法论（TOSS 与 TOSS-Pro）
核心思想是通过 Token 级的精细化筛选，精准切除有害 Token，同时保留对任务有益的 Token。

*   **TOSS 框架流程**：
    1.  **构建参考模型**：训练两个专门的 LoRA 模型：
        *   **安全性退化模型 ($f_{\theta_h}$)**：在有害数据集上训练，使其成为识别有害模式的“专家”。
        *   **效用导向模型 ($f_{\theta_u}$)**：在高质量效用数据上训练，代表下游任务的分布。
    2.  **Token 风险评估**：利用损失差异（Loss Difference）指标对自定义数据集中的每个 Token 进行评分：
        $$S(y) = -\log P(y|f_{\theta_u}) + \log P(y|f_{\theta_h})$$
        *评分逻辑*：如果一个 Token 在有害模型下损失低（概率高），而在效用模型下损失高，则其分值高，代表安全风险大。
    3.  **Token 级选择性微调**：根据全局评分排名，剔除得分最高的 $d\%$ 的 Token（通过 Mask 掩码实现），仅让安全的 Token 参与梯度更新。

*   **TOSS-Pro（渐进式增强）**：
    *   为了提高识别精度，引入迭代策略。每一轮利用当前评分选出最具代表性的有害样本来更新“安全性退化模型”，通过不断精炼该模型，使其能更敏锐地识别隐蔽的有害 Token。

### 6. 主要结论与发现
*   **性能超越**：TOSS 在安全性和效用之间取得了显著优于 SEAL 等样本级方法的平衡。在 Llama-3 上，其综合表现提升了约 20%。
*   **Token 级必要性**：实验证明有害信号和有用信号在样本内是交织的，简单的样本剔除会导致严重的效用损失。
*   **可迁移性**：在相同分词器（Tokenizer）的模型间，TOSS 选择的数据具有良好的迁移性，这意味着无需为每个模型重新计算评分。
*   **渐进式优势**：TOSS-Pro 通过迭代进一步提升了安全性，尤其是在处理隐蔽的有害数据时。

### 7. 优点与亮点
*   **粒度创新**：首次系统性地从 Token 级别诊断并解决微调安全性退化问题，打破了“样本为最小单位”的思维定式。
*   **双模型引导**：巧妙地结合了“有害专家”和“效用专家”的意见，通过损失差异实现精准打击。
*   **实用性强**：支持跨模型迁移，且基于 LoRA 的实现方式对计算资源的要求相对较低。

### 8. 不足与局限
*   **计算开销**：虽然使用了 LoRA，但相比于直接微调，TOSS 仍需要预先训练两个参考模型并对全量数据进行推理评分，增加了预处理阶段的时间成本。
*   **参考数据依赖**：参考模型的质量取决于预先选定的少量有害数据和效用数据，如果这些初始数据覆盖面不足，可能会影响评分的准确性。
*   **极端攻击防御**：虽然在常规微调攻击下表现优异，但对于极其隐蔽或新型的对抗性攻击，Token 级的损失差异是否依然敏感仍需进一步验证。
