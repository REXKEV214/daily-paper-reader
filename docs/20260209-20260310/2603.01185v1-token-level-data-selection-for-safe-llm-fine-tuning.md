## 摘要
在自定义数据集上对大语言模型（LLMs）进行微调已成为将这些模型适配到特定领域和应用的标准方法。然而，最近的研究表明，这种微调可能会导致模型安全性的显著下降。现有的防御方法在样本级别运行，通常在安全性和实用性之间面临难以令人满意的权衡。为了解决这一局限性，我们对微调过程中的安全性下降进行了系统的 Token 级诊断。基于此，我们提出了面向安全大语言模型微调的 Token 级数据选择（TOSS），这是一个通过测量安全性下降模型与实用性导向模型之间的损失差异来量化每个 Token 安全风险的新型框架。这种 Token 级的粒度能够准确识别并移除不安全的 Token，从而保留有价值的任务特定信息。此外，我们引入了一种渐进式细化策略 TOSS-Pro，该策略通过迭代增强安全性下降模型识别不安全 Token 的能力。大量的实验表明，我们的方法在微调过程中稳健地保护了大语言模型，同时实现了卓越的下游任务性能，显著优于现有的样本级防御方法。我们的代码可在 https://github.com/Polly-LYP/TOSS 获取。

## Abstract
Fine-tuning large language models (LLMs) on custom datasets has become a standard approach for adapting these models to specific domains and applications. However, recent studies have shown that such fine-tuning can lead to significant degradation in the model's safety. Existing defense methods operate at the sample level and often suffer from an unsatisfactory trade-off between safety and utility. To address this limitation, we perform a systematic token-level diagnosis of safety degradation during fine-tuning. Based on this, we propose token-level data selection for safe LLM fine-tuning (TOSS), a novel framework that quantifies the safety risk of each token by measuring the loss difference between a safety-degraded model and a utility-oriented model. This token-level granularity enables accurate identification and removal of unsafe tokens, thereby preserving valuable task-specific information. In addition, we introduce a progressive refinement strategy, TOSS-Pro, which iteratively enhances the safety-degraded model's ability to identify unsafe tokens. Extensive experiments demonstrate that our approach robustly safeguards LLMs during fine-tuning while achieving superior downstream task performance, significantly outperforming existing sample-level defense methods. Our code is available at https://github.com/Polly-LYP/TOSS.

#### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：在特定领域数据集上微调大语言模型（LLM）时，模型原本通过对齐（Alignment）建立的安全护栏极易遭到破坏，导致模型产生有害输出。
*   **研究背景**：现有的防御策略（如数据混合或样本级过滤）存在“粗粒度”缺陷。样本级过滤（如 SEAL）会将包含有用信息的整个样本剔除，导致模型在安全性和实用性（Utility）之间难以平衡。
*   **核心假设**：论文认为安全性退化的基本单位不是“样本”而是“Token”。有害信号往往集中在响应的初始部分或特定词汇中，而同一样本的其他 Token 可能对提升模型性能至关重要。

#### 2. 方法论：核心思想与技术细节
论文提出了 **TOSS (Token-level Data Selection for Safe LLM fine-tuning)** 框架，其核心流程如下：
*   **参考模型训练**：首先训练两个轻量化的参考模型（基于 LoRA）：
    1.  **安全性退化模型 ($f_{\theta_h}$)**：在有害数据集上微调，使其成为识别有害模式的“专家”。
    2.  **实用性导向模型 ($f_{\theta_u}$)**：在高质量任务数据上微调，代表下游任务的分布。
*   **Token 风险评估（损失差异指标）**：对于自定义数据集中的每个 Token，计算其在两个参考模型下的损失差异。
    *   **公式逻辑**：$Score = \text{Loss}(f_{\theta_u}) - \text{Loss}(f_{\theta_h})$。
    *   **含义**：如果一个 Token 在有害模型下损失低（容易预测）而在实用模型下损失高，则该 Token 被视为高风险。
*   **Token 级掩码与微调**：根据全局评分排名，剔除得分最高的 $d\%$（如 10%）的 Token，在微调过程中对这些 Token 进行掩码处理，使其不参与梯度更新。
*   **渐进式精炼 (TOSS-Pro)**：通过迭代策略，利用 TOSS 识别出的高风险样本不断更新“安全性退化模型”，从而更精准地识别隐蔽的有害 Token。

#### 3. 实验设计
*   **数据集与场景**：
    *   **自定义微调集**：REDORCA（包含 90k 条 SlimOrca 良性指令和 22k 条 Anthropic 红队有害指令）。
    *   **安全评测集**：HEx-PHI（11 类有害指令）、Anthropic HH 安全子集。
    *   **实用性评测集**：SlimOrca 测试集、GSM8K（数学推理）、MMLU/HellaSwag（通用能力）。
*   **对比方法**：
    *   标准 SFT（无防御）。
    *   Random（随机剔除 Token）。
    *   SafeInstr（混合安全指令）。
    *   DSIR（基于 n-gram 的重要性重采样）。
    *   **SEAL**（目前最先进的样本级双层优化选择方法）。
*   **评估指标**：使用 GPT-4o 作为裁判，计算相对于标准 SFT 的胜率（Win Rate），同时辅助使用攻击成功率（ASR）和有害性评分。

#### 4. 资源与算力
*   **微调技术**：统一使用 LoRA（rank=16），学习率 $1 \times 10^{-5}$，Batch Size 64，训练 2 个 Epoch。
*   **算力说明**：论文**未明确说明**具体的 GPU 型号（如 A100 或 H100）及总训练时长。但由于采用了 LoRA 且参考模型仅需在少量数据（3%-10%）上训练，推测其计算开销在可接受范围内。

#### 5. 实验数量与充分性
*   **实验规模**：
    *   涵盖了 Llama-2 (7B) 和 Llama-3 (8B, 3B, 1B) 多个模型系列。
    *   进行了跨模型迁移实验（在 8B 模型上选的数据直接用于 1B/3B 模型）。
    *   **消融实验**：对比了全局排名 vs 局部排名、Token 级 vs 样本级、不同参考模型组合的贡献、迭代次数的影响。
    *   **鲁棒性分析**：测试了不同剔除比例（0% 到 100%）以及在纯良性数据下的表现。
*   **充分性评价**：实验设计非常系统且严谨，通过多维度对比验证了 Token 级选择的必要性，实验结果具有较强的说服力。

#### 6. 主要结论与发现
*   **Token 级防御更优**：TOSS 在安全性和实用性上均显著优于样本级方法（SEAL），平均性能提升约 16%-20%。
*   **位置敏感性**：诊断分析发现，有害信号虽集中在响应的前几个 Token，但中间和末尾也存在散布的风险点，固定位置掩码会损害实用性。
*   **迁移性强**：在相同分词器（Tokenizer）的模型间，TOSS 选择的数据具有良好的通用性，无需为每个模型重新计算。
*   **渐进式提升**：TOSS-Pro 通过迭代优化参考模型，能进一步挖掘隐蔽的有害模式，提升防御上限。

#### 7. 优点与亮点
*   **粒度创新**：首次系统性地从 Token 级别诊断并解决微调安全退化问题，打破了“样本为最小单位”的思维定式。
*   **损失差异指标**：巧妙地结合了安全与实用两个维度的参考模型，实现了“外科手术式”的精准剔除。
*   **实用性保护**：相比于直接拒绝或删除样本，TOSS 保留了样本中的良性部分，极大缓解了过度拒绝（Over-refusal）问题。

#### 8. 不足与局限
*   **额外开销**：虽然是 LoRA，但仍需预先训练两个参考模型并对全量数据进行推理评分，增加了微调前的预处理成本。
*   **分词器依赖**：该方法高度依赖分词器的一致性，若基础模型分词器发生变化，数据选择结果不可复用。
*   **极端场景挑战**：对于极其隐晦的逻辑陷阱（而非显式有害词汇），Token 级的损失差异是否依然敏感，仍需在更复杂的对抗场景下验证。
*   **超参数敏感性**：剔除比例 $d$ 的选择需要根据数据集质量进行调整，缺乏自动化的动态确定机制。
