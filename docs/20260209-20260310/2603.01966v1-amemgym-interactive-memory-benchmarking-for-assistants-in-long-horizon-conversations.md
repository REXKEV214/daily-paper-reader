## 摘要
用户与基于大语言模型（LLM）的助手之间的长程交互需要有效的记忆管理，然而当前的方法在记忆的训练和评估方面面临挑战。现有的记忆基准测试依赖于静态的离策（off-policy）数据作为上下文，限制了评估的可靠性和可扩展性。为了填补这些空白，我们推出了 AMemGym，这是一个支持记忆驱动个性化的在策（on-policy）评估与优化的交互式环境。AMemGym 采用结构化数据采样来预定义用户画像、依赖状态的问题以及状态演变轨迹，从而能够低成本地生成高质量且与评估对齐的交互。LLM 模拟的用户通过角色扮演暴露潜在状态，同时保持结构化状态的一致性。基于结构化数据的全面指标指导着助手的评估与优化。广泛的实验揭示了现有记忆系统（如 RAG、长上下文 LLM 和智能体记忆）之间的性能差距及其原因。AMemGym 不仅能够有效筛选竞争方法，还可能推动记忆管理策略的自我进化。通过将结构化状态演变与自由形式的交互相结合，我们的框架为提升对话智能体的记忆能力提供了一个可扩展且诊断信息丰富的环境。

## Abstract
Long-horizon interactions between users and LLM-based assistants necessitate effective memory management, yet current approaches face challenges in training and evaluation of memory. Existing memory benchmarks rely on static, off-policy data as context, limiting evaluation reliability and scalability. To address these gaps, we introduce AMemGym, an interactive environment enabling on-policy evaluation and optimization for memory-driven personalization. AMemGym employs structured data sampling to predefine user profiles, state-dependent questions, and state evolution trajectories, enabling cost-effective generation of high-quality, evaluation-aligned interactions. LLM-simulated users expose latent states through role-play while maintaining structured state consistency. Comprehensive metrics based on structured data guide both assessment and optimization of assistants. Extensive experiments reveal performance gaps in existing memory systems (e.g., RAG, long-context LLMs, and agentic memory) and corresponding reasons. AMemGym not only enables effective selection among competing approaches but also can potentially drive the self-evolution of memory management strategies. By bridging structured state evolution with free-form interactions, our framework provides a scalable, diagnostically rich environment for advancing memory capabilities in conversational agents.

#### 1. 核心问题与整体含义（研究动机和背景）
在开发基于大语言模型（LLM）的助手时，实现长程对话能力（即跨多轮对话管理和利用记忆）至关重要。然而，现有的记忆基准测试（如 MSC, LongMemEval 等）大多依赖**静态、离策（Off-policy）**的数据。
*   **局限性**：离策评估无法捕捉助手自身对话选择带来的后果，导致评估结果不真实，且可能误导记忆策略的优化。
*   **动机**：需要一个**在策（On-policy）**的交互式环境，能够模拟真实用户动态揭露信息的过程，并提供可扩展、可诊断的评估框架。

#### 2. 方法论：核心思想与关键技术
AMemGym 引入了一个交互式环境，通过“逆向工程”策略将自由对话锚定在结构化数据上：
*   **结构化数据生成**：
    1.  **用户画像采样**：从 Nemotron-Personas 抽取 100K 画像。
    2.  **状态模式（Schema）定义**：根据评估问题提取关键状态变量（如：导师辅导方式、烹饪时间偏好等）及其可能的取值范围。
    3.  **状态演变轨迹**：模拟用户状态随时间（多个周期）的演变，并生成相应的“生活事件”叙述。
    4.  **个性化答案生成**：为每个状态组合预生成唯一的正确答案，并通过 LLM 反思步骤确保答案的唯一性和无歧义性。
*   **在策交互**：使用 LLM 模拟用户进行角色扮演，根据预定义的轨迹在对话中自然地“暴露”潜在状态。
*   **诊断性指标**：除了总分（Accuracy）和归一化记忆得分外，将失败归因为三个阶段：**写入（Write）**、**读取（Read）**和**利用（Utilization）**。

#### 3. 实验设计
*   **数据集/场景**：包含 20 个独特用户画像，每个画像 10 个评估问题。分为 **Base**（10 个演变周期，需 128K+ 上下文）和 **Extra**（20 个周期，需 512K+ 上下文）两种配置。
*   **对比方法**：
    *   **原生 LLM**：包括 Claude 4, Gemini 2.5/3, GPT-4.1/5.1/5.2, DeepSeek-v3, Qwen3 等。
    *   **记忆系统**：标准 RAG、智能体写入（外部索引 AWE）、智能体写入（在策上下文 AWI）。
    *   **开源框架**：Mem0, Nemori, A-Mem。
*   **自我进化实验**：测试智能体是否能根据环境反馈（Factual Feedback）自主优化其记忆写入提示词（Prompt）。

#### 5. 实验数量与充分性
*   **实验规模**：涵盖了 200 个核心评估问题，在 11-21 个不同的时间点进行测试，总计产生数千次交互。
*   **充分性与客观性**：
    *   进行了**元评估（Meta-evaluation）**：通过人工标注验证了状态暴露的清晰度（99.1%）和对话一致性（99.2%）。
    *   对比了在策与离策的排名差异，证明了在策评估的必要性。
    *   测试了不同模拟用户模型（如用 DeepSeek-v3 替代 GPT-4.1）对结果的影响，验证了框架的鲁棒性。实验设计考虑了公平性，所有记忆系统使用相同的底层生成模型。

#### 6. 主要结论与发现
*   **在策评估的必要性**：离策评估会产生“重用偏差”，导致记忆系统的排名与实际交互表现不符。
*   **长上下文挑战**：即使是支持超长上下文的顶级模型（如 Gemini 2.5），在长程交互后的记忆得分也会大幅下降，甚至接近随机猜测。
*   **智能体记忆优于原生模型**：精心设计的智能体记忆系统（如 AWE）通过选择性地提取和存储信息，显著优于直接依赖长上下文或标准 RAG。
*   **自我进化可行性**：智能体能够通过环境提供的失败反馈，自主将模糊的记忆指令优化为具体的、可操作的规则，从而降低“写入失败率”。

#### 7. 优点
*   **交互性与真实性**：填补了静态基准测试无法模拟动态交互后果的空白。
*   **细粒度诊断**：通过“写、读、用”三阶段指标，能够精准定位记忆系统的薄弱环节。
*   **可扩展性**：全自动的数据生成流程允许低成本地构建针对不同领域、不同难度的测试集。
*   **闭环优化**：不仅是评估工具，还提供了支持智能体策略自我进化的训练环境。

#### 8. 不足与局限
*   **模拟器偏差**：评估质量高度依赖于作为“用户模拟器”的 LLM 的角色扮演能力，可能存在模型固有的偏见。
*   **状态离散化**：目前的状态演变基于离散的变量和取值，可能无法完全覆盖现实世界中极其复杂、连续或模糊的信息变化。
*   **API 依赖**：大规模运行该基准测试仍需支付一定的商业 API 费用，对于资源受限的团队可能存在门槛。
*   **场景局限**：目前主要侧重于个性化助手场景，对于协作任务或多智能体博弈等更复杂的记忆需求覆盖较少。
