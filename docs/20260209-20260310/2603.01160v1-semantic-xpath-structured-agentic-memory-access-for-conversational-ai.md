## 摘要
对话式人工智能（ConvAI）智能体越来越多地维护结构化记忆，以支持长期的、面向任务的交互。上下文内记忆（In-context memory）方法将不断增长的历史记录附加到模型输入中，这在上下文窗口限制下扩展性较差。基于检索增强生成（RAG）的方法检索与请求相关的信息，但大多数方法假设记忆集合是扁平的，并忽略了结构。我们提出了 Semantic XPath，这是一个树状结构的记忆模块，用于访问和更新结构化对话记忆。Semantic XPath 相比扁平 RAG（flat-RAG）基准性能提升了 176.7%，同时仅使用了上下文内记忆所需 token 数量的 9.1%。我们还介绍了 SemanticXPath Chat，这是一个端到端的 ConvAI 演示系统，可可视化结构化记忆和查询执行细节。总之，本文展示了构建在结构化记忆之上的下一代长期、面向任务的 ConvAI 系统的一个候选方案。

## Abstract
Conversational AI (ConvAI) agents increasingly maintain structured memory to support long-term, task-oriented interactions. In-context memory approaches append the growing history to the model input, which scales poorly under context-window limits. RAG-based methods retrieve request-relevant information, but most assume flat memory collections and ignore structure. We propose Semantic XPath, a tree-structured memory module to access and update structured conversational memory. Semantic XPath improves performance over flat-RAG baselines by 176.7% while using only 9.1% of the tokens required by in-context memory. We also introduce SemanticXPath Chat, an end-to-end ConvAI demo system that visualizes the structured memory and query execution details. Overall, this paper demonstrates a candidate for the next generation of long-term, task-oriented ConvAI systems built on structured memory.

### 2. 方法论：Semantic XPath
论文提出了 **Semantic XPath**，这是一种将传统的 XML 路径查询语言（XPath）与现代 LLM 的语义能力相结合的树状记忆模块。

*   **核心思想**：将对话记忆建模为**有根树（Rooted Tree）**，并定义一套扩展的查询语法，允许智能体通过路径导航和语义过滤来精确定位内存子结构。
*   **关键技术细节**：
    *   **数据模型**：记忆由节点（Node）组成，每个节点有类型（如 `Day`, `POI`）和属性。支持版本控制（Version），记录修改历史。
    *   **查询语法**：
        *   **轴导航（Axis Navigator）**：`/` 选择子节点，`//` 选择所有后代。
        *   **节点选择器（Node Selector）**：按类型筛选。
        *   **语义相关性算子（Semantic Relevance Operator）**：使用 `~=` 进行语义匹配（如 `[node ~= "conference"]`），支持聚合函数（avg, max 等）和逻辑运算。
    *   **算法流程**：
        1.  **翻译**：LLM 将用户自然语言请求翻译为 Semantic XPath 查询语句。
        2.  **执行**：通过递归评估函数 $E(Q, W)$ 在树结构上运行。它结合了**结构匹配**（路径导航）和**语义评分**（计算节点与查询字符串的相似度或蕴含度）。
        3.  **输出**：返回加权的节点集合，供下游 LLM 生成最终回复或执行内存更新。
