## 摘要
代理式 AI 需要持久化内存来存储超出大语言模型（LLM）有限上下文窗口的用户特定历史记录。现有的内存系统使用稠密向量数据库或知识图谱遍历（或两者混合），导致高检索延迟和较差的存储可扩展性。我们推出了 Hippocampus，这是一种代理式内存管理系统，它使用紧凑的二进制签名进行语义搜索，并使用无损 Token-ID 流进行精确的内容重构。其核心是动态小波矩阵（DWM），它对这两种流进行压缩和协同索引，以支持在压缩域中的超快速搜索，从而避免了昂贵的稠密向量或图计算。该设计随内存大小线性扩展，使其适用于长周期的代理部署。实验表明，我们的评估显示 Hippocampus 将端到端检索延迟降低了高达 31 倍，并将单次查询的 Token 占用减少了高达 14 倍，同时在 LoCoMo 和 LongMemEval 基准测试中保持了准确性。

## Abstract
Agentic AI require persistent memory to store user-specific histories beyond the limited context window of LLMs. Existing memory systems use dense vector databases or knowledge-graph traversal (or hybrid), incurring high retrieval latency and poor storage scalability. We introduce Hippocampus, an agentic memory management system that uses compact binary signatures for semantic search and lossless token-ID streams for exact content reconstruction. Its core is a Dynamic Wavelet Matrix (DWM) that compresses and co-indexes both streams to support ultra-fast search in the compressed domain, thus avoiding costly dense-vector or graph computations. This design scales linearly with memory size, making it suitable for long-horizon agentic deployments. Empirically, our evaluation shows that Hippocampus reduces end-to-end retrieval latency by up to 31$\times$ and cuts per-query token footprint by up to 14$\times$, while maintaining accuracy on both LoCoMo and LongMemEval benchmarks.

### 1. 核心问题与整体含义（研究动机和背景）
代理式 AI（Agentic AI）在处理长程任务时，需要持久化内存来存储超出大语言模型（LLM）上下文窗口的历史信息。然而，现有的内存系统面临以下挑战：
*   **效率低下**：基于检索增强生成（RAG）的稠密向量搜索或基于知识图谱（KG）的遍历，在存储和检索时会产生极高的延迟和计算开销。
*   **可扩展性差**：随着记忆增长，向量计算和图查询的成本呈非线性增加。
*   **Token 浪费**：频繁的总结和多级检索消耗了大量昂贵的 LLM Token。
**研究动机**：开发一种轻量级、原生支持压缩且可线性扩展的内存基座，以支持智能体在长周期部署中的快速写入和低延迟召回。

### 2. 方法论：核心思想与关键技术
Hippocampus 抛弃了传统的“Token 中心、嵌入重型”设计，转而采用**双重表示策略**：
*   **核心思想**：利用**动态小波矩阵（Dynamic Wavelet Matrix, DWM）**同时索引“无损 Token-ID 流”（用于精确重建）和“二进制签名流”（用于语义搜索）。
*   **关键技术细节**：
    1.  **动态小波矩阵 (DWM)**：这是对静态小波矩阵的改进，支持在不重新构建整个索引的情况下进行增量追加（Append-only）。它通过 `rank`（计数）、`select`（定位）和 `access`（访问）原语，在压缩域内直接进行位级操作。
    2.  **随机索引（Random Indexing）语义哈希**：将高维向量投影到随机超平面，生成紧凑的二进制签名。语义相似的 Token 会生成汉明距离较小的签名。
    3.  **汉明球搜索（Hamming-ball Search）**：检索时，将查询关键词转化为签名，利用 CPU 原生的位运算（XOR 和 POPCOUNT）在签名 DWM 中快速查找汉明半径内的候选者。
    4.  **两阶段流程**：
        *   **构建**：Token 化 -> 生成二进制签名 -> 更新内容 DWM 和签名 DWM。
        *   **查询**：提取关键词 -> 签名匹配（签名 DWM）-> 获取元数据指针 -> 精确重建内容（内容 DWM）。

### 6. 主要结论与发现
*   **性能飞跃**：相比于 MemGPT 等系统，Hippocampus 将端到端检索延迟降低了高达 **31 倍**。

*   **线性扩展**：DWM 的设计保证了系统随记忆容量增长仍能保持高效。

### 7. 优点（亮点）
*   **位级检索创新**：将传统的向量相似度计算转化为极快的位运算，极大地提升了检索吞吐量。
*   **无损重建**：与总结式记忆不同，它能通过内容 DWM 100% 还原原始 Token，避免了信息丢失。
*   **冷启动友好**：构建过程无需调用 LLM，显著降低了系统部署和维护的经济成本。

### 8. 不足与局限
*   **语义表达上限**：二进制签名虽然高效，但在处理极度复杂的语义细微差别时，可能不如高维稠密向量精确（尽管实验显示差距较小）。

*   **应用限制**：目前主要针对文本/Token 流，对于多模态（如图像、视频）记忆的直接支持尚需进一步扩展。
