## 摘要
智能体 AI 需要持久化记忆来存储超出大语言模型（LLM）有限上下文窗口的用户特定历史记录。现有的记忆系统使用稠密向量数据库或知识图谱遍历（或两者混合），导致高检索延迟和较差的存储可扩展性。我们推出了 Hippocampus，这是一种智能体记忆管理系统，它使用紧凑的二进制签名进行语义搜索，并使用无损 Token-ID 流进行精确的内容重构。其核心是一个动态小波矩阵（DWM），它对这两种流进行压缩和协同索引，以支持在压缩域中的超快速搜索，从而避免了昂贵的稠密向量或图计算。该设计随记忆容量线性扩展，使其适用于长周期的智能体部署。实验表明，我们的评估显示 Hippocampus 将端到端检索延迟降低了高达 31 倍，并将单次查询的 Token 占用减少了高达 14 倍，同时在 LoCoMo 和 LongMemEval 基准测试中保持了准确性。

## Abstract
Agentic AI require persistent memory to store user-specific histories beyond the limited context window of LLMs. Existing memory systems use dense vector databases or knowledge-graph traversal (or hybrid), incurring high retrieval latency and poor storage scalability. We introduce Hippocampus, an agentic memory management system that uses compact binary signatures for semantic search and lossless token-ID streams for exact content reconstruction. Its core is a Dynamic Wavelet Matrix (DWM) that compresses and co-indexes both streams to support ultra-fast search in the compressed domain, thus avoiding costly dense-vector or graph computations. This design scales linearly with memory size, making it suitable for long-horizon agentic deployments. Empirically, our evaluation shows that Hippocampus reduces end-to-end retrieval latency by up to 31$\times$ and cuts per-query token footprint by up to 14$\times$, while maintaining accuracy on both LoCoMo and LongMemEval benchmarks.

*   **基于 RAG/向量数据库：** 插入时需要昂贵的嵌入（Embedding）生成，检索时涉及高维向量相似度计算，延迟高且 Token 开销大。

**核心目标：** 开发一种轻量级、原生支持压缩、且能实现超快速检索和精确重建的记忆模块，以支持长周期部署的智能体。

### 2. 方法论
**核心思想：**
Hippocampus 摒弃了传统的稠密向量表示，采用**双重表示策略**，并利用**动态小波矩阵（Dynamic Wavelet Matrix, DWM）**进行协同索引。

**关键技术细节：**
1.  **双重表示：**
    *   **内容 DWM（Content DWM）：** 将原始文本序列转化为 Token-ID 流，以位图形式存储，支持无损的内容重建。
    *   **签名 DWM（Signature DWM）：** 利用**随机索引（Random Indexing）**技术将 Token 及其上下文转化为紧凑的二进制签名（Binary Signatures），用于语义搜索。
2.  **动态小波矩阵（DWM）：**
    *   这是对传统静态小波矩阵的改进，支持**追加（Append）**操作，适合智能体不断产生的对话流。
    *   支持三种核心原语：`access(i)`（检索位置 i 的符号）、`rank(c, i)`（统计符号 c 在前缀中出现的次数）、`select(c, j)`（查找符号 c 第 j 次出现的位置）。
3.  **检索流程：**
    *   **关键词提取：** LLM 从查询中提取关键词。
    *   **汉明球搜索（Hamming-ball Search）：** 将关键词转化为二进制签名，在 Signature DWM 中通过位运算（XOR 和 POPCOUNT）快速查找汉明距离在阈值内的候选位置。
    *   **精确重建：** 根据候选位置的元数据，从 Content DWM 中提取原始 Token-ID 并解码回文本。

### 7. 优点
*   **架构创新：** 首次将小波矩阵这一简洁数据结构引入智能体记忆管理，实现了“压缩域搜索”。
*   **端到端优化：** 不仅优化了“读”（检索），还通过免嵌入设计彻底解决了“写”（存储）的高开销问题。
*   **硬件友好：** 核心搜索逻辑基于位运算（Bitwise operations），能够极大地发挥现代 CPU/GPU 的指令集优势。
*   **无损重建：** 与摘要式记忆不同，它能精确还原原始对话 Token。

### 8. 不足与局限
*   **语义表达上限：** 二进制签名虽然高效，但在处理极度复杂的语义细微差别时，可能不如 4096 维的浮点向量精确（尽管实验显示差距很小）。
*   **关键词依赖：** 检索的第一步依赖于 LLM 提取关键词，如果关键词提取不准，后续的汉明搜索可能会漏掉相关信息。
*   **冷启动/字典依赖：** 系统依赖于预定义的 Token 字典，对于包含大量 OOV（词汇表外）词汇的特殊领域可能需要额外处理。
*   **应用限制：** 目前主要针对文本 Token，尚未扩展到多模态（如图像、音频）记忆的直接索引。
