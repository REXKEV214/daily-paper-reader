## 摘要
Engram 条件记忆通过将静态知识查找与动态计算解耦，已成为大语言模型（LLM）中一个极具前景的组件。由于 Engram 表现出稀疏的访问模式并支持预取，其庞大的嵌入表非常适合卸载到低层级内存中。在本文中，我们提出使用 Compute Express Link (CXL) 内存池进行 Engram 存储。与 RDMA 相比，CXL 提供了 Engram 极小且离散的检索模式所需的细粒度、低延迟访问。我们将基于 CXL 的 Engram 池集成到 SGLang 中，实现了接近 DRAM 的端到端性能。这为未来集成 Engram 的 LLM 提供了一种可扩展且具有成本效益的存储解决方案，且不会损害推理性能。

## Abstract
Engram conditional memory has emerged as a promising component for LLMs by decoupling static knowledge lookup from dynamic computation. Since Engram exhibits sparse access patterns and supports prefetching, its massive embedding tables are well-suited for offloading to lower-tier memory. In this paper, we propose using Compute Express Link (CXL) memory pool for Engram storage. Compared to RDMA, CXL provides fine-grained and low-latency access required by minimal and discrete retrieval patterns of Engram. We integrate the CXL-based Engram pool into SGLang, achieving near-DRAM end-to-end performance. This provides a scalable and cost-efficient storage solution for future Engram-integrated LLMs without compromising inference performance.

以下是对该论文的结构化总结：

### 1. 核心问题与研究动机
*   **核心问题**：随着 LLM 规模扩大，DeepSeek 提出的 Engram 架构（通过 N-Gram 嵌入表实现静态知识查找）虽然能提升模型性能，但其庞大的嵌入表（可达数百 GB）导致了严重的“显存墙”问题。
*   **研究动机**：Engram 的访问模式具有**稀疏性**（每 token 仅需几 KB）、**只读性**和**延迟容忍性**（可异步预取）。传统的 RDMA 存储池在处理这种细粒度、离散的小数据包时延迟过高，而本地 DRAM 扩展成本巨大。因此，研究者寻求一种低延迟、高性价比的池化存储方案。

### 2. 方法论
*   **核心思想**：利用 CXL 2.0/3.0 协议的内存池化特性，将 Engram 嵌入表卸载到独立的 CXL 存储节点，通过 CXL 交换机供多个计算节点共享。
*   **关键技术细节**：
    *   **CXL 内存池化**：使用 CXL 交换机连接计算服务器和 CXL 内存扩展卡，提供硬件级的 load/store 指令支持，绕过网络协议栈。
    *   **优化访问例程**：
        *   **CXL → CPU**：利用 OpenMP 多线程并行执行 `memcpy`，实现高并发读取。
        *   **CXL → GPU**：开发自定义 CUDA 核函数，通过 `cudaHostRegister` 将 CXL 内存映射为 GPU 可访问的地址，实现点对点（P2P）数据传输，完全绕过 CPU 拷贝。
    *   **框架集成**：在 SGLang 推理框架中实现了初始化（单 rank 加载）、异步预取（在计算前层 Transformer 时提前拉取 Engram 数据）和计算融合。

### 6. 主要结论与发现
*   **延迟优势**：CXL 的检索延迟比 RDMA 低 1-2 个数量级，且非常接近本地 DRAM，能够满足 Engram 预取的时间窗口要求。
*   **端到端性能**：在 SGLang 中，基于 CXL 的方案实现了接近原生 DRAM 的推理吞吐量（性能损失极小）。
*   **成本效益**：随着节点数和模型规模增加，CXL 池化方案的成本优势显著。例如，对于 400B 的 Engram 表，在 16 个节点规模下，CXL 方案可比本地 DRAM 方案节省约 16.6 万美元。

*   **首创性**：首次将 CXL 内存池技术应用于 LLM 的 Engram 条件记忆优化。

*   **实用性强**：直接集成在主流推理框架 SGLang 中，证明了方案的工程可行性。

### 8. 不足与局限
*   **模型权重限制**：由于缺乏开源的 Engram 模型权重，实验结果基于模拟计算，可能与真实模型的数值分布存在微小偏差。
