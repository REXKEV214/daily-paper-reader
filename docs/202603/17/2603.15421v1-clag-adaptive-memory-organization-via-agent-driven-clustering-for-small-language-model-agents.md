## 摘要
大语言模型智能体高度依赖外部记忆来支持知识复用和复杂推理任务。然而，大多数记忆系统将经验存储在单一的全局检索池中，这可能会逐渐稀释或破坏存储的知识。这一问题在小语言模型（SLM）中尤为突出，因为它们极易受到无关上下文的影响。我们提出了 CLAG，一种基于聚类的智能体记忆框架，SLM 智能体通过聚类主动组织记忆。CLAG 采用 SLM 驱动的路由器将输入的记忆分配到语义连贯的簇中，并自主生成簇特定的配置文件（包括主题摘要和描述性标签），从而将每个簇构建为自包含的功能单元。通过在这些结构化邻域内进行局部演化，CLAG 有效减少了跨主题干扰并增强了内部记忆密度。在检索过程中，该框架采用两阶段流程，首先通过配置文件过滤相关簇，从而排除干扰项并缩小搜索空间。在多个问答数据集上使用三种 SLM 骨干进行的实验表明，CLAG 在保持轻量和高效的同时，在答案质量和鲁棒性方面始终优于先前的智能体记忆系统。

## Abstract
Large language model agents heavily rely on external memory to support knowledge reuse and complex reasoning tasks. Yet most memory systems store experiences in a single global retrieval pool which can gradually dilute or corrupt stored knowledge. This problem is especially pronounced for small language models (SLMs), which are highly vulnerable to irrelevant context. We introduce CLAG, a CLustering-based AGentic memory framework where an SLM agent actively organizes memory by clustering. CLAG employs an SLM-driven router to assign incoming memories to semantically coherent clusters and autonomously generates cluster-specific profiles, including topic summaries and descriptive tags, to establish each cluster as a self-contained functional unit. By performing localized evolution within these structured neighborhoods, CLAG effectively reduces cross-topic interference and enhances internal memory density. During retrieval, the framework utilizes a two-stage process that first filters relevant clusters via their profiles, thereby excluding distractors and reducing the search space. Experiments on multiple QA datasets with three SLM backbones show that CLAG consistently improves answer quality and robustness over prior memory systems for agents, remaining lightweight and efficient.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：传统的智能体记忆系统通常将所有交互记录存储在一个“全局检索池”中。随着记忆量的增加，这种无结构的存储方式会导致两个问题：
    1.  **检索干扰**：搜索空间扩大，增加了检索到语义相似但任务无关（噪声）记忆的概率。
    2.  **知识稀释**：在全局范围内进行记忆演化（如总结、更新）会导致不同主题的知识相互干扰。
*   **背景**：这一问题在**小语言模型（SLM）**中尤为严重，因为 SLM 对无关上下文的鲁棒性较差，极易受到干扰信息的影响。
*   **研究目标**：通过引入“聚类”思想，将记忆组织成语义连贯的局部邻域，从而减少干扰并提高 SLM 智能体在长程任务中的表现。

### 2. 方法论：核心思想与关键技术
CLAG 的核心思想是将聚类视为一种由智能体驱动的动态操作，而非静态的预处理步骤。其架构包含三个关键组件：
*   **智能体驱动路由（Agentic Routing）**：
    *   当新记忆进入时，系统先通过向量距离筛选出候选簇。
    *   随后由 SLM 智能体根据各簇的“画像”（Profile，包含摘要和标签）决定将其分配到哪个簇。
    *   **自适应机制**：如果新记忆与现有簇相似度过低，则创建新簇；如果某个簇超过预设大小（如 300 条），则通过 K-Means 自动分裂。
*   **局部化演化（Localized Evolution）**：
    *   记忆的更新、链接生成和压缩仅在所属的簇内进行。
    *   这种方式模仿了人类的认知模式（Schema），即新信息只修正相关的知识结构，而不干扰无关领域。
*   **簇感知两阶段检索（Cluster-Aware Two-Stage Retrieval）**：
    *   **第一阶段**：智能体根据查询（Query）从候选簇画像中筛选出相关的簇。
    *   **第二阶段**：仅在选定的簇内进行细粒度的记忆检索。这极大地缩小了搜索空间并排除了跨主题的干扰项。

*   **性能提升**：CLAG 在所有数据集和模型上均优于基准方法，尤其在 **BioASQ** 这种噪声大、术语多的场景下，性能提升最为显著（F1 分数提升巨大）。
*   **SLM 友好性**：实验证明，通过结构化管理记忆，SLM 可以在不增加参数量的情况下，展现出接近大模型的抗干扰能力。

### 7. 优点
*   **结构化创新**：将无序的全局记忆池转变为有序的语义簇，有效解决了长程记忆中的“干扰”难题。
*   **局部演化**：通过限制更新范围，既保证了记忆的密度，又降低了计算复杂度。
*   **鲁棒性强**：两阶段检索机制为 SLM 提供了一个天然的“过滤器”，显著提升了在复杂上下文中的回答质量。

### 8. 不足与局限
*   **Prompt 敏感性**：系统高度依赖于 SLM 生成簇画像和进行路由决策的 Prompt 质量，不同模型的 Prompt 兼容性可能存在差异。
*   **分布偏移风险**：如果智能体遇到的新话题与旧话题界限模糊，路由决策可能会出现偏差。
*   **隐私与治理**：论文虽提到这属于记忆组织机制，但在实际应用中，持久化存储用户记忆涉及隐私和数据保留政策的挑战。
*   **模型规模限制**：虽然在 SLM 上表现优异，但在超大规模模型（如 70B+）上，这种结构化带来的边际收益可能会因为大模型本身极强的抗干扰能力而减小。
