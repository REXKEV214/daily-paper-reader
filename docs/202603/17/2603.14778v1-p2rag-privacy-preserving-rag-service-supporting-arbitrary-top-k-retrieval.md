## 摘要
检索增强生成 (RAG) 使大语言模型能够利用外部知识，但外包 RAG 服务会引发数据所有者和用户的隐私担忧。隐私保护 RAG 系统通过执行安全 Top-$k$ 检索来解决这些问题，这通常涉及通过安全排序来识别相关文档。然而，现有系统在支持任意 $k$ 值方面面临挑战，原因包括无法更改 $k$、新的安全问题或在大 $k$ 值下的效率下降。这是一个重大限制，因为现代长上下文模型通常在检索集较大时获得更高的准确率。我们提出了 $p^2$RAG，一种支持任意 Top-$k$ 检索的隐私保护 RAG 服务。与现有系统不同，$p^2$RAG 避免了对候选文档进行排序。相反，它使用交互式二分法来确定 Top-$k$ 文档集。在安全性方面，$p^2$RAG 在两个半诚实且互不串通的服务器上使用秘密共享，以保护数据所有者的数据库和用户的提示词。它通过强制执行限制和验证来防御恶意用户，并严格限制数据库的信息泄露。实验表明，在 $k = 16$ 到 $1024$ 的情况下，$p^2$RAG 比最先进的 PRAG 快 3--300 倍。

## Abstract
Retrieval-Augmented Generation (RAG) enables large language models to use external knowledge, but outsourcing the RAG service raises privacy concerns for both data owners and users. Privacy-preserving RAG systems address these concerns by performing secure top-$k$ retrieval, which typically is secure sorting to identify relevant documents. However, existing systems face challenges supporting arbitrary $k$ due to their inability to change $k$, new security issues, or efficiency degradation with large $k$. This is a significant limitation because modern long-context models generally achieve higher accuracy with larger retrieval sets. We propose $p^2$RAG, a privacy-preserving RAG service that supports arbitrary top-$k$ retrieval. Unlike existing systems, $p^2$RAG avoids sorting candidate documents. Instead, it uses an interactive bisection method to determine the set of top-$k$ documents. For security, $p^2$RAG uses secret sharing on two semi-honest non-colluding servers to protect the data owner's database and the user's prompt. It enforces restrictions and verification to defend against malicious users and tightly bound the information leakage of the database. The experiments show that $p^2$RAG is 3--300$\times$ faster than the state-of-the-art PRAG for $k = 16$--$1024$.

### 1. 核心问题与研究背景
*   **研究动机**：检索增强生成（RAG）虽然解决了大语言模型（LLM）的幻觉和时效性问题，但将数据外包给 RAG 服务商会带来严重的隐私风险（包括数据所有者的私有数据库和用户的查询提示词）。
*   **核心问题**：现有的隐私保护 RAG 系统通常采用“安全排序”或“安全 Top-$k$ 检索”技术。这些技术在处理较大的 $k$ 值（如 $k=1024$）时，要么效率极低，要么不支持动态调整 $k$。而现代长上下文模型在检索集较大时往往能获得更高的准确率，因此亟需一种能高效支持任意、大 $k$ 值的隐私保护方案。

### 2. 方法论：核心思想与技术细节
$p^2$RAG 的核心思想是**放弃昂贵的安全排序，转而采用交互式二分法（Interactive Bisection）**来确定 Top-$k$ 文档集。
*   **系统架构**：采用两台半诚实且互不串通的服务器（Two-server model）。数据所有者和用户通过**秘密共享（Secret Sharing）**将数据和查询向量分布在两台服务器上。
*   **关键技术流程**：
    1.  **距离计算**：服务器在秘密共享状态下计算查询向量与数据库中所有文档向量的点积（余弦相似度）。
    2.  **交互式二分查找**：用户与服务器进行多轮交互。用户提出一个距离阈值 $d_k$，服务器利用**分布式比较函数（DCF）**统计距离小于该阈值的文档数量 $c$，并将 $c$ 返回给用户。用户根据 $c$ 与目标 $k$ 的关系调整阈值，直到找到包含约 $k$ 个文档的范围。
    3.  **安全验证与限制**：为防止恶意用户通过多次查询窃取数据，系统限制了二分查找的迭代次数（$O(\log N)$）和最终返回的文档数量上限。
    4.  **文本检索**：确定索引后，用户通过单服务器私有信息检索（PIR）获取加密文档，服务器会验证用户检索的索引是否在二分查找确定的范围内。

### 3. 实验设计
*   **数据集**：使用 BEIR 评测基准中的 **trec-covid** 数据集（包含 171,332 个文档）。
*   **对比方法（Benchmark）**：与当前最先进的隐私保护检索系统 **PRAG** 进行对比。
*   **评估指标**：计算准确性（Recall）、检索相关性得分（Relevance Score）、服务器处理时间（Latency）以及通信开销（Communication Volume）。

### 4. 资源与算力
*   **硬件环境**：实验在一台配备两颗 **AMD EPYC 7352 CPU**（共 96 个逻辑核心）和 188 GiB 内存的物理服务器上运行。

### 5. 实验数量与充分性
*   **实验规模**：
    *   测试了不同的文档规模 $N$（从 $2^{14}$ 到 $2^{20}$）。
    *   测试了广泛的检索数量 $k$（从 16 到 1024）。
    *   进行了准确性验证，证明了向量量化（从浮点数到 64 位整数）对召回率几乎没有影响。
*   **充分性评价**：实验设计较为充分，覆盖了从小型到百万级规模的数据库，并重点展示了在大 $k$ 值下的性能优势。对比实验在相同硬件环境下进行，具有客观性。

### 6. 主要结论与发现
*   **性能飞跃**：在 $k=16$ 到 $1024$ 的区间内，$p^2$RAG 的处理速度比 PRAG 快 **3 到 300 倍**。
*   **大 $k$ 优势**：与传统方法随 $k$ 增大而性能剧降不同，$p^2$RAG 的二分查找开销与 $k$ 的相关性较低，在大 $k$ 场景下表现极其稳定。
*   **高准确性**：尽管引入了整数截断和二分近似，但在实际数据集上的召回率仍接近 1.0，且检索到的文档相关性得分很高。

### 7. 优点与亮点
*   **算法创新**：巧妙地将 Top-$k$ 问题转化为阈值查找问题，避开了安全排序这一性能瓶颈。
*   **灵活性**：支持用户在查询时动态指定任意 $k$ 值，无需重新预处理数据库。
*   **安全性平衡**：在提供高效检索的同时，通过严格限制迭代次数和结果集大小，将数据库的信息泄露量控制在可证明的极低范围内（$O(\log^2 N)$ 比特）。

### 8. 不足与局限
*   **交互开销**：二分查找需要用户与服务器之间进行多轮通信（RTT 较多），在网络延迟极高的环境下可能会影响用户体验。
*   **安全假设**：依赖于“两台服务器不串通”的假设，如果两台服务器被同一实体控制，隐私保护将失效。
*   **恶意用户泄露**：虽然泄露量被限制，但恶意用户仍能通过统计信息获得关于数据库分布的微量知识（物理泄露）。
*   **存储压力**：秘密共享和 DCF 密钥会产生一定的存储和通信冗余，对于超大规模数据库（如十亿级），内存压力会显著增大。
