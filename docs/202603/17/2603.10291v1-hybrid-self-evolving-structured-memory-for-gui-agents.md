## 摘要
视觉语言模型（VLM）的显著进展使得 GUI 智能体能够以类人的方式与计算机进行交互。然而，由于长程工作流、多样化的界面以及频繁的中间错误，现实世界的计算机使用任务仍然具有挑战性。先前的研究通过大量轨迹集合为智能体构建外部记忆，但依赖于对离散摘要或连续嵌入的扁平化检索，缺乏人类记忆的结构化组织和自演化特性。受大脑启发，我们提出了混合自演化结构化记忆（HyMEM），这是一种基于图的记忆，将离散的高层符号节点与连续的轨迹嵌入相结合。HyMEM 维持一种图结构，以支持多跳检索、通过节点更新操作实现的自演化，以及推理过程中的即时工作记忆刷新。大量实验表明，HyMEM 持续提升了开源 GUI 智能体的性能，使 7B/8B 参数规模的主干网络能够媲美或超越强大的闭源模型；值得注意的是，它将 Qwen2.5-VL-7B 的性能提升了 22.5%，并优于 Gemini2.5-Pro-Vision 和 GPT-4o。

## Abstract
The remarkable progress of vision-language models (VLMs) has enabled GUI agents to interact with computers in a human-like manner. Yet real-world computer-use tasks remain difficult due to long-horizon workflows, diverse interfaces, and frequent intermediate errors. Prior work equips agents with external memory built from large collections of trajectories, but relies on flat retrieval over discrete summaries or continuous embeddings, falling short of the structured organization and self-evolving characteristics of human memory. Inspired by the brain, we propose Hybrid Self-evolving Structured Memory (HyMEM), a graph-based memory that couples discrete high-level symbolic nodes with continuous trajectory embeddings. HyMEM maintains a graph structure to support multi-hop retrieval, self-evolution via node update operations, and on-the-fly working-memory refreshing during inference. Extensive experiments show that HyMEM consistently improves open-source GUI agents, enabling 7B/8B backbones to match or surpass strong closed-source models; notably, it boosts Qwen2.5-VL-7B by +22.5% and outperforms Gemini2.5-Pro-Vision and GPT-4o.

#### 1. 核心问题与整体含义（研究动机和背景）
当前的 GUI（图形用户界面）智能体在处理长程工作流、多样化界面及频繁的中间错误时表现欠佳。虽然现有研究尝试为智能体配备外部记忆，但大多采用**扁平化检索**（基于离散摘要或连续嵌入的相似度匹配），这与人类大脑结构化、自演化的记忆机制相去甚远。人类记忆能将多模态细节（海马体）与高层概念/策略（新皮质）结合，并随新经验不断更新。本文旨在构建一种受脑科学启发的**混合自演化结构化记忆（HyMEM）**，以提升 GUI 智能体在复杂任务中的规划与执行能力。

#### 2. 方法论：核心思想与关键技术
HyMEM 的核心是一个**混合图结构记忆系统**，结合了离散符号节点与连续轨迹嵌入：
*   **混合图模式（Graph Schema）**：
    *   **高层策略节点（Discrete）**：存储启发式策略（如“按价格从低到高过滤”）。
    *   **中层属性节点（Discrete）**：语义标签（如 #搜索, #过滤）。
    *   **底层轨迹节点（Continuous）**：使用连续嵌入保留细粒度的多模态证据。
*   **自演化构建机制**：
    *   **冗余检查与更新**：当新轨迹进入时，通过 VLM 判断其信息增量，执行 **ADD（添加）**、**MERGE（合并）** 或 **REPLACE（替换）** 操作，确保记忆库在不无限膨胀的前提下持续进化。
*   **推理时的记忆利用**：
    *   **结构化检索**：采用“种子+扩展”策略，先检索相似节点，再通过图边扩展至相关策略节点，增加检索的深度与多样性。
    *   **即时工作记忆刷新（On-the-fly Refresh）**：在执行过程中，VLM 会检测任务阶段的转换（Phase Shift），动态更新当前的工作记忆，确保智能体在长程任务中不迷失方向。

#### 5. 实验数量与充分性
*   **实验规模**：在 3 个主流基准、多个垂直领域（购物、地图、旅游、百科）进行了广泛测试。
*   **消融实验**：详细分析了全局演化（Global Evolution）、局部刷新（Local Refresh）、记忆库规模（从 500 到 8000 条轨迹）以及检索策略（相似度 vs. 多样性平衡）的影响。
*   **客观性**：实验设置了统一的环境、任务采样和评估标准（LLM-as-Judge），并对比了多种不同架构的开源与闭源模型，实验设计较为客观、公平。

#### 6. 主要结论与发现
*   **性能飞跃**：HyMEM 显著提升了开源模型表现。Qwen2.5-VL-7B 在加入 HyMEM 后，成功率从 12.5% 提升至 **35.0%（+22.5%）**。
*   **超越闭源**：搭载 HyMEM 的 7B/8B 规模模型在平均性能上**超越了 GPT-4o 和 Gemini 2.5 Pro**。
*   **自演化的有效性**：全局演化通过合并冗余信息实现了记忆的“亚线性增长”，而局部刷新有效解决了长程任务中的上下文漂移问题。
*   **混合优势**：实验证明，单纯的离散或连续记忆均不如两者结合的混合模式，混合模式在策略抽象和感知接地（Grounding）之间达到了最佳平衡。

#### 7. 优点（亮点）
*   **脑启发架构**：巧妙地模拟了海马体与新皮质的分工，解决了 GUI 任务中“高层规划”与“底层操作”脱节的问题。
*   **图结构检索**：突破了传统的扁平化向量检索，通过图扩展实现了更具逻辑关联性的经验提取。
*   **高效演化**：引入了基于 VLM 的冗余检查机制，使记忆库具备自我清理和优化的能力，提高了存储效率。

#### 8. 不足与局限
*   **更新策略依赖启发式**：目前的 ADD/MERGE/REPLACE 决策依赖于 VLM 的启发式判断，而非通过强化学习等更系统的方法进行端到端优化。
*   **模型规模限制**：受限于算力，论文未在更大规模（如 32B 或 70B）的模型上验证 HyMEM 的扩展性。
*   **实时性开销**：推理时的“即时刷新”和“图扩展检索”可能会增加单步决策的延迟和 API 调用成本。
