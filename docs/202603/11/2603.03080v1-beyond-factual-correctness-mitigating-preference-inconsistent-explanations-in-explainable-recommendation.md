## 摘要
基于大语言模型（LLM）的可解释推荐系统能够生成流畅且事实正确的解释，但仍可能利用与用户历史偏好相冲突的属性来为物品辩护。这种偏好不一致的解释虽然逻辑上成立，但推理过程缺乏说服力，且现有的标准幻觉或忠实度指标很大程度上无法检测到此类问题。我们对这种失效模式进行了形式化定义，并提出了 PURE，一个遵循“先选择后生成”范式的偏好感知推理框架。PURE 不仅仅改进生成环节，还干预证据选择过程。在用户意图、特异性和多样性的指导下，它选择一组紧凑的、以物品为中心的多跳推理路径，这些路径既有事实依据，又与用户偏好结构相一致，从而抑制通用且个性化程度较低的证据。随后，通过保留关系约束的结构感知提示，将选定的证据注入 LLM 生成过程中。为了衡量偏好不一致性，我们引入了一种以用户为中心的特征级评估指标，该指标能够揭示基于事实性的度量标准所忽略的失配问题。在三个真实世界数据集上的实验表明，PURE 在保持具有竞争力的推荐准确度、解释质量和推理效率的同时，持续减少了偏好不一致的解释和事实幻觉。这些结果强调，值得信赖的解释不仅需要事实上的正确性，还需要与用户偏好相一致的论证。

## Abstract
LLM-based explainable recommenders can produce fluent explanations that are factually correct, yet still justify items using attributes that conflict with a user's historical preferences. Such preference-inconsistent explanations yield logically valid but unconvincing reasoning and are largely missed by standard hallucination or faithfulness metrics. We formalize this failure mode and propose PURE, a preference-aware reasoning framework following a select-then-generate paradigm. Instead of only improving generation, PURE intervenes in evidence selection, it selects a compact set of multi-hop item-centric reasoning paths that are both factually grounded and aligned with user preference structure, guided by user intent, specificity, and diversity to suppress generic, weakly personalized evidence. The selected evidence is then injected into LLM generation via structure-aware prompting that preserves relational constraints. To measure preference inconsistency, we introduce a feature-level, user-centric evaluation metric that reveals misalignment overlooked by factuality-based measures. Experiments on three real-world datasets show that PURE consistently reduces preference-inconsistent explanations and factual hallucinations while maintaining competitive recommendation accuracy, explanation quality, and inference efficiency. These results highlight that trustworthy explanations require not only factual correctness but also justification aligned with user preferences.

### 1. 核心问题与整体含义（研究动机和背景）
*   **研究动机**：现有的基于大语言模型（LLM）的可解释推荐系统主要关注“事实正确性”（Factual Correctness），即解释是否符合物品的客观属性。然而，作者发现了一个被忽视的失效模式——**偏好不一致（Preference-Inconsistency）**。
*   **核心问题**：系统生成的解释可能在事实上是正确的，但其辩护理由却与用户的历史偏好相悖。例如，向一位喜欢“温馨治愈”电影的用户推荐《肖申克的救赎》时，系统若强调其“残酷的监狱生活和痛苦挣扎”，虽然属实，但无法说服该用户。
*   **研究意义**：这种偏好不一致会侵蚀用户信任。现有的幻觉检测指标无法识别此类“事实正确但理由错误”的问题，因此需要一种新的机制来确保解释既保真又对齐用户偏好。

### 2. 方法论：PURE 框架
PURE（Preference-aligned Unhallucinated Reasoning for Explanation）采用“先选择后生成”的范式，核心步骤如下：

*   **结构化语义索引（Offline）**：利用知识图谱（KG）构建多跳（1-3跳）推理路径。使用关系图注意力网络（RGAT）预先计算节点和关系的结构化嵌入，并存入向量数据库。
*   **目标感知的用户意图建模**：不使用静态的用户画像，而是以候选推荐物品为查询（Query），通过注意力机制动态聚合用户历史行为，生成与当前推荐相关的“目标感知意图向量”。
*   **特异性感知路径剪枝（Specificity-Aware Pruning）**：
    *   定义了**节点特异性评分 $I(v, u)$**，结合了结构特异性（惩罚高频中心节点）、语义特异性（通过聚类熵衡量）和偏好相关性。
    *   通过该评分过滤掉通用但无个性化信息的路径（如“这是一部热门电影”），保留与用户意图高度对齐的推理路径。
*   **多样性感知重排**：使用最大边界相关性（MMR）算法对路径进行重排，确保解释涵盖导演、题材、风格等多个维度，避免语义冗余。
*   **偏好感知生成**：
    *   **结构感知子图编码**：使用 Graph Transformer 编码选定的推理路径。
    *   **混合提示策略**：将子图编码转化为“软提示词”（Soft Prompts），与文本形式的“硬提示词”（Hard Prompts）共同输入 LLM，通过 LoRA 微调生成最终解释。

*   **偏好一致性至关重要**：实验证明，即使事实完全正确的解释，如果偏好不一致，也会被人类判定为低质量。

### 7. 优点与亮点
*   **新颖的视角**：首次正式定义并量化了“偏好不一致解释”这一失效模式，超越了单纯的“事实正确性”。

*   **干预前置**：不同于大多数研究在生成阶段做文章，PURE 强调在“证据选择”阶段进行干预，从源头上解决了通用性证据泛滥的问题。

### 8. 不足与局限
*   **知识图谱依赖**：该方法高度依赖外部知识图谱的质量和覆盖率。对于 KG 中缺失的冷门物品，效果可能会打折扣。

*   **动态性挑战**：目前主要基于静态历史记录，对于用户兴趣的瞬时漂移（如临时帮朋友买书）的捕捉能力尚待进一步研究。
