## 摘要
大语言模型 (LLMs) 通常需要微调 (FT) 才能在下游任务中表现出色，但即使训练数据集仅包含良性数据，微调也可能引发安全对齐偏移。先前的研究表明，引入极小比例的有害数据就会显著损害 LLM 的拒绝行为，导致其遵循有害请求。现有的防御方法通常依赖于全模型范围的干预，例如限制参数更新或注入额外的安全数据，这可能会限制通用性并降低下游任务的性能。为了解决这些局限性，我们提出了一种名为“通过约束 Token 保持安全对齐”(PACT) 的微调框架，该框架可以稳定模型在安全 Token 上的置信度。我们的方法基于一项经验观察，即安全对齐行为体现在模型的 Token 级输出置信度中，且通常集中在极少数安全相关的 Token 上。在下游微调期间，我们在每个响应步骤中对微调模型进行正则化，使其在安全相关 Token 上的置信度与已对齐的参考模型保持一致，同时让非安全 Token 基本上不受约束，以确保有效的任务适配。这种有针对性的约束在防止对齐偏移的同时，避免了通常会牺牲模型效用的全局性限制。

## Abstract
Large language models (LLMs) often require fine-tuning (FT) to perform well on downstream tasks, but FT can induce safety-alignment drift even when the training dataset contains only benign data. Prior work shows that introducing a small fraction of harmful data can substantially compromise LLM refusal behavior, causing LLMs to comply with harmful requests. Existing defense methods often rely on model-wide interventions, such as restricting which parameters are updated or injecting additional safety data, which can limit generality and degrade downstream task performance. To address these limitations, we propose a fine-tuning framework called Preserving Safety Alignment via Constrained Tokens (PACT), which stabilizes the model's confidence on safety tokens. Our approach is motivated by the empirical observation that safety-aligned behavior is reflected in the model's token-level output confidence and is often concentrated on a small subset of safety-related tokens. During downstream fine-tuning, we regularize the fine-tuned model to match the aligned reference model's confidence on safety-related tokens at each response step, while leaving non-safety tokens largely unconstrained to allow effective task adaptation. This targeted constraint prevents alignment drift without imposing global restrictions that typically trade off with model utility.

## 1. 核心问题与整体含义
**研究动机与背景：**
大语言模型（LLMs）在通过微调（Fine-tuning, FT）适配下游任务时，普遍面临“安全对齐漂移”（Safety Alignment Drift）的问题。研究发现，即使微调数据完全良性，或者仅包含极少量（如0.1%）的有害数据，模型原本通过强化学习（RLHF）获得的拒绝有害请求的能力也会迅速瓦解。

**核心问题：**
现有的防御手段（如限制参数更新范围或注入安全补丁数据）往往过于粗犷，容易导致模型在下游任务上的性能（Utility）受损。本文旨在寻找一种更精细的干预机制，在不牺牲模型任务效能的前提下，低成本地锁住模型的安全防线。

## 2. 方法论：PACT 框架
论文提出了 **PACT (Preserving safety Alignment via Constrained Tokens)** 框架，其核心思想是：**安全对齐行为高度集中在极少数关键 Token 的输出置信度上**（如 "I", "cannot", "sorry" 等）。

**关键技术细节：**
1.  **安全 Token 识别：** 通过“教师强制”（Teacher-forcing）机制，对比已对齐模型（$M_{safe}$）与基座模型（$M_{base}$）在面对有害提示词时输出概率的差异。计算每个 Token 的全局差异得分，选出前 $K$ 个（实验中 $K=50$）作为安全 Token 集合。
2.  **加权正则化（Weighted Regularization）：** 在微调的损失函数中引入一个针对安全 Token 的加权 KL 散度项。该项强制微调模型在这些特定 Token 上的预测概率分布与原始对齐模型保持一致，而对其他非安全 Token 则不予约束。
3.  **安全信号校准（Calibration of Safety Signal）：** 为了防止微调中的有害前缀干扰参考模型的判断，PACT 结合了两种视图：
    *   **全上下文视图：** 参考模型看到完整的 Prompt 和 Response。
    *   **无提示词视图：** 参考模型仅看到 Assistant 的响应部分。
    *   通过一个动态门控系数（基于概率分散程度）自动混合这两个视图，确保在有害语境下依然能提供纯净的“拒绝”信号。

## 6. 主要结论与发现
1.  **极少数 Token 决定安全性：** 仅需约束前 50 个安全相关的 Token，就能在很大程度上防止模型在微调中“变坏”。
2.  **安全与效能可兼得：** PACT 在大幅降低攻击成功率（ASR）的同时，下游任务的准确率（Acc）几乎与标准微调持平，显著优于 SafeLoRA 等方法。
3.  **有害前缀的干扰：** 证实了在微调过程中，有害的上下文会抑制参考模型的安全信号，而通过“无提示词视图”校准是解决这一问题的关键。
4.  **泛化性强：** 该方法在不同架构（Qwen, Llama, Gemma）和不同中毒比例下均表现稳健。

## 7. 优点（亮点）
*   **高杠杆效应：** 抓住“少量 Token”这一关键变量，实现了以小博大的防御效果。
*   **非侵入性：** 不需要修改模型架构，也不需要像传统方法那样注入大量的安全训练样本。
*   **动态校准：** 巧妙地解决了参考模型在有害上下文下“自身难保”的逻辑悖论。
*   **任务适配性好：** 由于约束范围极窄，模型在学习新知识时受到的阻力极小。

## 8. 不足与局限
